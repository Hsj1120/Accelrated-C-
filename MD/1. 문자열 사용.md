# 1 문자열 사용

## 1.1 입력

Hello, world! 프로그램을 특정 사람에게 인사하는 프로그램으로 수정할 수 있다

```c++
// 이름을 묻고, 인사를 건넴

#include <iostream>
#include <string>

int main()
{
    std::cout << "Please enter your first name : ";
    std::string name;
	std::cin >> name;

	// 인사말 작성
	std::cout << "Hello, " << name << "!" << std::endl;
	return 0;
}
```

프로그램을 실행시키면 다음과 같은 메세지를 출력한다.
 ```
 Please enter your first name : 
 ```

Hans라고 입력하면 다음과 같은 메세지를 출력한다.

```
Hello, Hans!
```

입력한 내용을 전달받으려면 이를 저장할 장소가 필요하며 그러한 장소를 **변수**라고 하며 변수는 이름이 있는 **객체**이다.

객체는 타입이 정해진 컴퓨터 메모리의 일부이다.

변수를 사용하려면 구현체에 변수의 이름과 타입을 알려줘야 한다.

프로그렘에서 변수 이름은 name이고 타입은 std::string이다.

std:: 뒤에 오는 이름 string은 프로그래밍 언어의 기본 영역이 아닌 표준 라이브러리의 일부라는 뜻이며 std::string에 관련된 헤더인 <string>이  있으므로 #include를 추가하였다



첫번째 실행문을 살펴보면

```c++
std::cout << "Please enter your first name : ";
```

사용자의 이름을 묻는 메세지를 출력한다. 여기에는 조작어 std::endl이 존재하지 않아 메세지를 출력한 후 행바꿈을 하지 않는다.



다음 실행문을 살펴보면

```c++
std::string name
```

이것은 std::string타입의 변수 name의 **정의문**이며 함수 본문에 있으므로 name은 중괄호 안에만 존재하는 **지역 변수**이다.

컴퓨터가 '}'에 도달하면 변수 name은 **소멸**하고 변수가 차지하던 메모리를 반환하다. 즉 지역 변수의 수명이 제한적이므로 구별해줘야 한다.

객체의 타입에는 **인터페이스** 개념이 포함되어 있다.

인터페이스는 어떤 타입의 객체로 할 수 있는 동작들의 모음이며, 예를 들어 문자열 변수 name의 정의문은 문자열 (string)에서 사용하는 모든 라이브러리를 name에서 다룰 것임을 의미한다

그리고 name의 동작 중 하나는 문자열을 **초기화**하는 것이 있으며, 문자열 변수를 정의하면 암묵적으로 변수가 초기화 된다.

이유는 표준라이브러리에서는 모든 문자열 객체가 값을 갖는다고 정의해 두었기 때문이다.

원하는 값을 넣으면 문자열을 만들 수 있지만 이 동작이 없으면 문자열은 문자를 가지고 있지 않고 이러한 문자열을 

**'빈 문자열'** 또는 '**null 문자열**'이라고 한다.



```c++
std::cin >> name;
```

입력 시 >> 연산자와 std::cin을 사용한다.

이 예제에서 >>연산자는 표준 입력으로 문자열을 읽어서 name이라는 객체에 저장한다.

문자열을 읽도록 라이브러리에 요청하면 **공백문자**(스페이스, 탭, 백스페이스, 행바꿈)을 무시하고 다음 공백 문자나 **EOF**이 나타날 때까지 모든 문자들을 읽기 시작하여 변수 name에 넣는다.

따라서 std::cin >> name을 실행하면 표준 입력에서 문자열을 읽고 문자열을 구성하는 문자들을 name에 저장한다.



입력동작에서 부수적으로 발생하는 결과로 사용자의 이름을 묻는 메세지가 출력장치에 표시된다.

일반적으로 입출력 라이브러리는 출력작업의 최적화를 위해 **버퍼**라고 하는 내부 데이터 구조를 사용한다.

라이브러리는 버퍼를 사용하여 출력할 문자들을 모으고, 필요할 때만 버퍼의 저장한 내용을 출력하고 버퍼를 지운다.

이 방법으로 여러 번의 출력 작업을 효율적으로 처리한다.



버퍼를 비우는 상황은 3가지가 있다.

1. 버퍼가 가득찰 경우 라이브러리는 자동으로 버퍼를 지운다
2. 라이브러리가 표준 입력 스트림으로 입력 내용을 요청받으면 버퍼가 가득찰 때까지 기다리지 않고 비운다.
3. 개발자가 명시한 코드로 비운다.

예제 프로그램의 경우 cout을 사용하여 메세지를 출력할 때, 출력 내용은 표준 출력 스트림과 관련된 버퍼의 저장되고, 그다음 cin을 사용하여 입력을 받는데, 이 작업은 cout 버퍼를 지우므로 화면에 출력할 수 있다.

이어지는 출력문에서는 Hello 다음에 name의 값, 마지막으로 std::endl을 작성하는데 std::endl 값이 출력되면 이 행에서 해야할 작업이 끝나므로 버퍼를 비운다.

이 때 시스템은 버퍼의 내용을 출력 스트림에 강제로 즉시 반영한다.



## 1.2 테두리

```c++
// 이름을 입력 받아 테두리로 묶인 인사말을 생성
#include <iostream>
#include <string>

int main()
{
    std::cout << "Please enter your first name : ";
    std::string name;
	std::cin >> name;

	//출력하려는 메시지를 구성
	const std::string greeting = "Hello, " + name + "!";

	//인사말의 두번째 행과 4번째 행
	const std::string spaces(greeting.size(), ' ');
	const std::string second = "* " + spaces + " *";

	//인사말의 첫번째 행과 다섯번째 행
	const std::string first(second.size(), '*');
	
	// 모두 출력
	std::cout << std::endl;
	std::cout << first << std::endl;
	std::cout << second << std::endl;
	std::cout << "* " << greeting << " *" << std::endl;
	std::cout << second << std::endl;
	std::cout << first << std::endl;
	return 0;
}


```

이 프로그램을 실행하면 

```
Please enter your first name : HANS

****************
*              *
* Hello, HANS! *
*              *
****************
```

프로그램 코드를 분석해 보자

먼저 프로그램은 사용자의 이름을 입력 받아 변수 name에 넣고, 그런 다음에 출력하려는 메세지가 들어 있는 상수 greeting을 정의한다.

다음으로 greeting의 문자 갯수 만큼 공백을 넣는 상수 spaces를 정의한다.

그리고 변수 second의 문자 개수만큼 *를 포함하는 상수 first를 정의한다.

마지막으로 한 번에 항 행씩 인사말을 출력한다.



새로운 개념 3가지를 정리해보면

첫번째 개념은 **변수를 정의하면서 변수의 값을 넣을 수 있다**는 것이다.

변수 이름과 세미콜론 사이에 = 기호와 변수에 넣을 값을 명시하면 된다.

변수의 값이 서로 다른 타입이면 구현체는 넣은 값을 변수의 타입으로 **반환**한다.



두번째 개념은 +기호를 사용하여 1개의 문자열과 1개읠 문자열 리터럴을 **결합**하거나 2개의 문자열끼리 결합할 수 있다.

2개의 문자열 리터럴 사이에는 기호를 사용한 결합관계가 성립되지 않는다.

'+' 연산자는 피연산자의 타입에 따라 실행하는 연산이 다르다.

3+4의 연산과  "Hello, " + name + "!";의 연산은 다르게 동작하는데 다양한 타입의 피연산자에 따라 다른 의미로 도악하는 것을 연산자의 **오버로드**라고 한다.



세번째 개념은 상수를 정의할 때 사용하는 **const**이다.

const는 소멸할 때까지 값을 변경하지 않겠다는 약속이다.

const로 정의한 상수는 나중에 값을 변경하지 못하므로 정의할 때 반드시 초기화 해야 한다.

초깃값이 그 자체로 상수일 필요가 없으며 예제 프로그램에서는 name 갑을 입력한 뒤에 greeting 값을 결정한다.

name의 값은 프로그램을 실행하기 전까지 정해저 있지 않으므로 name은 상수가 아니다.



```c++
const std::string spaces(greeting.size(), ' ');
```

상수 greeting을 정의할 때 = 기호를 사용해서 초기화 하였다.

=를 사용하면 변수가 가져야 하는 값을 명시적으로 지시할 수 있는데, 상수 spaces는 쉼표로 구분하고 괄호로 묶인 2개의 표현식을 거쳐 상수를 생성하도록 지시한다.

상수나 변수의 생성 방법은 타입에 따라 다르다. 무엇으로 문자열을 만드는 지 어떤 의미가 있는지 살펴보자.



첫번째 표현식 greeting.size()는 **멤버함수**를 호출하는 방법이다.

결론적으로 greeting이라는 객체는 size라는 구성요소가 있으며, 이 구성요소는 함수이므로 값을 얻으려고 호출할 수 있다.

상수 greeting은 std::string 타입이며 greeting.size()로 greeting의 문자 개수를 나타내는 정수를 얻을 수 있다.



두번째 표현식 ' '은 **문자 리터럴**이다.

문자 리터럴은 문자열 리터럴과는 완전히 다른데 문자 리터럴은 작은따옴표로 묶고 문자열 리터럴은 큰따옴표로 묶는다.

문자 리터럴은 기본 타입이 char이다. 문자 리터럴은 단일 문자를 나타내며, 문자열 리터럴 안에는 특수한 의미가 있는 문자는 문자는(\\t, \\n 등등) 문자 리터럴과 같은 의미이다.



마지막으로 spaces 정의를 살펴보면, 정수값과 문자값을 받아 정수값만큼의 횟수로 문자값을 복사해 문자열을 만든다.

예를 들어 

```
std::string stars (10, *)
```

라고 정의하면 stars의 size는 10이고 stars는

```
**********
```

값을 가진다.

따라서 spaces에는 greeting과 같은 개수의 공백문자가 들어간다.



## 1.3 핵심 정리

### 문자 타입

- char : 구현체가 정의한 기본 타입이며 일반 문자를 지정한다.
- wchar_t : 한국어 같은 언어의 문자를 표현할 만큼 크기가 큰 '확장 문자'를 저장하는 기본 타입이다.



### 문자열 타입

표준 헤더인 <string>에 정의되어 있다.

문자열 타입 객체는 0개 이상의 문자로 이루어진 순차열을 갖으며 정수를 n , 문자를 c, 입력 스트림을 is, 출력 스트림을 os라고 한다면 문자열 연산의 종류는 다음과 같다

- std::string s : std::string 타입의 변수 s를 정의하며 변수 s는 빈 문자열로 초기화된다.
- std::string t = s : std::string 타입의 변수 t를 정의하며 변수 t는 s에 있는 문자들의 복사본으로 초기화 된다.
- std::string z(n,c) : std::string 타입의 변수 z를 정의하며 변수 z는 문자 c의 복사본 n개를 포함하여 초기화된다.
- os << s : os가 가리키는 출력 스트림에 s에 포함된 문자들을 서식 변경 없이 출력한다.
- is >> s : 공백 외의 문자가 등장할 때까지 is가 가리키는 스트림에서 문자들을 읽고 버린다. 그 다음 is가 가리키는 스트림에서 새로운 공백이 들어올 때까지 연속된 문자들을 s로 읽는다.
- s + t : 이 표현식의 결과는 s에 있는 문자들의 복사본과 t에 있는 문자들의 복사본을 결합한 std::string의 객체이다 s 또는 t 중 하나가 문자열 리터럴 이거나 char 타입의 값일 수 있다.
- s.size() : s에 있는 문자들의 갯수이다.



### 변수

변수는 다음 세 가지 방법 중 하나로 정의할 수 있다.

1. std::string hello = "Hello" : 초기값을 명시하여 변수를 정의
2. std::string stars(100, '*') : 타입과 주어진 표현식에 따라 변수를 생성
3. std::string name; : 타입에 따라 암묵적인 초기화로 변수를 정의

한쌍의 중괄호 내부에 정의된 변수는 중괄호 내부 코드를 실행할 때만 존재하며 구현체가 }에 도달하면 구현체는 변수를 소멸시키고 변수가 차지하던 메모리를 시스템에 반환한다.



const로 상수를 정의하면 소멸할 때 까지 값을 바꾸지 않겠다는 의미이다.



### 입력

std::cin >> v 를 실행하면 표준 입력 스트림에 있는 모든 공백을 무시한 후에 표준 입력의 내용을 변수 v에 넣는다.

연속적인 입력 동작을 하려는 목적으로 istream 타입인 std::cin 을 반환한다
