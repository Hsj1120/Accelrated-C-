# 4. 프로그램 및 데이터 구조화

C++ 라이브러리의 기능들은 다음과 같은 공통적인 특징이 있다.

- 특정 문제를 해결한다.
- 다른 기능들과 연계하지 않고 독립적이다.
- 기능의 이름이 있다.

지금까지 구현한 프로그램에서는 첫번째 특징 외의 나머지 두가지 특징이 없다. 

작은 프로그램에서는 문제가 되지 않지만 더 큰 문제를 해결하는 경우 이름이 있는 독립적인 부분으로 프로그램을 나누지 않으면 관리하기 어렵다.

이번 챕터에서는 함수와 데이터 구조를 사용해서 프로그램을 구조화하는 방법을 학습 후 개별적으로 컴파일 할 수 있도록 프로그램을 다수의 파일로 나눌 것이다.



## 4.1 연산 구조화

한 과목에서 중간시험 20%, 기말 시험 40%, 평균 과제 점수 40%에 가중치를 주어 최종 점수를 구하는 함수를 만들어 보자

연산 하나를 프로그램 몇몇 지점으로 반복실행해야 한다면 해당 연산을 함수로 만들 것을 고려해야 한다. 함수를 사용하면 코드양이 줄어들 뿐만 아니라 수정하기도 편한다.

성적을 계산하는 함수는 다음과 같다.

```c++
double grade(double midterm, double final, double homework)
{
	return 0.2 * midterm + 0.4 * final + 0.4 * homework;
}
```

함수는 반환 타입, 함수 이름, 소괄호로 묶인 **매개변수 목록**,중괄호로 묶인 함수의 본문 순서대로 작성한다.

매개변수는 midterm, final, homework 모두 double 타입이며 매개변수는 함수 안에서 지역 변수인 것처럼 동작한다. 즉 호출할 때 생성되고 반환할 때 소멸된다

다른 변수들과 마찬가지로 매개변수는 사용하기 전에 정의해야한다. 하지만 매개변수는 다른 변수들과 달리 정의하는 즉시 만들어지지 않고 함수를 호출할 때만 만들어 진다. 따라서 함수를 호출할 때마다 매개변수를 초기화하는 데 사용하는 **인수**를 제공해야 한다

최종 점수를 구하는 코드를 다음처럼 작성할 수 있다.

```c++
cout << "your final grade is " << setprecision(3)
    << grade(midterm, final, sum / count)
    << setprecision(prec) << endl;
```

호출하는 함수의 매개변수에 맞게 인수를 제공해야할 뿐만 아니라 순서도 같아야 한다.

따라서 grade 함수를 호출할 때 첫 번째 인수는 중간시험 점수, 두 번째 인수는 기말시험 점수, 세번째 인수는 종합과제 점수여야 한다.

인수는 sum / count 처럼 변수가 아닌 표현식으로도 사용할 수 있다. 일반적으로 인수 각각은 대응하는 매개변수를 초기화 하는 데 사용하며 함수 내부에서 지역변수처럼 동작한다. 예를 들어 grade(midterm, final, sum / count)를 호출하면 grade 함수의 매개변수들은 인수 값을 직접 참조하지 않고 인수 값의 복사본으로 초기화한다. 

매개변수가 인수 값의 복사본을 갖는 이러한 동작을 **값에 의한 호출**이라고 한다.



### 4.1.1 median 함수

벡터의 중앙값을 구하는 부분을 다른 관점으로 살펴볼 것이며 중앙값을 구하는 부분을 조금 바꿔보자

```c++
double median(vector<double> vec)
{
	typedef vector<double>::size_type vec_sz;
	vec_sz size = vec.size();
	if (size == 0)
		throw domain_error("median of an empty vector");
	sort(vec.begin(), vec.end());
	vec_sz mid = size / 2;
	return size % 2 == 0 ? (vec[mid] + vec[mid - 1]) / 2 : vec[mid];
}
```

벡터의 이름을 homework가 아닌 vec라고 정했다. 즉 median 함수는 과제 점수의 중앙값이 아니라 모든 벡터의 중앙 값을 구할 수 있다는 의미이다.

vec_sz는 다른 목적으로 해당이름을 사용할 때 충돌을 피하려고 지역 타입으로 정의한다.



눈에 띄는 변화는 벡터가 비었을 때의 동작이다. 이럴 때는 프로그램을 실행하는 사람에게 문제가 있음을 알려야 한다.

이번에는 누가 어떤 목적으로 프로그램을 사용하는지 모르므로 더 일반적은 문제 알림 방식이 필요하다. 즉, 벡터가 비었을 때 **예외를 던지는 것**이다.

프로그램이 예외를 던지면 throw가 있는 부분에서 실행을 멈춘 후, 호출자에게 전달할 정보가 있는 **예외 객체**와 함께 프로그램의 또 다른 부분으로 이동한다.

앞 코드에서 던진 예외는 domain_error이며 <stdexcept>에서 정의한 형식이다.

던지려는 domain_error 객체를 생성할 때 무엇이 잘못되었는지 설명하는 문자열을 던질 수 있다.



함수를 호출하는 것은 인수를 매개변수에 복사하는 것이기도 하다. 따라서 median 함수를 호출할 때는 인수르 사용한 벡터를 vec로 복사한다.

median함수라면 시간이 걸리더라도 인수를 매개변수에 복사하는 것이 유용하다. sort 함수를 호출하여 매개변수의 값을 바꾸기 때문이다.

인수를 복사하는 것은 sort 함수로 바꾸는 사항들이 호출자에 영향을 미치지 않도록 한다. 벡터의 중앙값을 구하는 것이 벡터 자체를 바꾸면 안되기 때문이다.



### 4.1.2 성적 산출 방식 다시 구현하기

방금 코드를 바탕으로 종합 과제 점수를 구하는 부분을 함수 내부에 다시 작성하면

```c++
double grade(double midterm, double final, const vector<double>& hw)
{
	if (hw.size() == 0)
		throw domain_error("student has done no homework");

	return grade(midterm, final, median(hw));
}
```

3가지 흥미로운 부분이 있다.



첫번째는 인수로 지정한 const vector<double>& hw인데 이 타입은 "const <double> 벡터를 참조"한다고 한다.

참조한다는 것은 해당 이름이 또 다른 호칭이라는 의미이기도 한다. 예를 들면 

```c++
vector<double> homework;
vector<double>& hw = homework; // hw는 homework의 동의어
```

hw는 homework에 또 다른 이름이다. hw로 실행하는 모든 작업은 homework와 같은 작업을 한다는 의미이다. 반대일 때도 마찬가지다. const를 추가하게 되면

```c++
// chw는 조회만 가능한 homework의 동의어
const vector<double>& chw = homework;
```

chw는 homework에 또다른 이름이지만 const를 사용했으므로 chw로 값을 바꾸는 어떠한 작업도 실행하지 않겠다는 약속이다.

참조는 원본에 또 다른 이름이다. 참조의 참조라는 것은 무의미 하다. 참조의 참조를 정의하는 것은 원본 객체의 참조를 정의하는 것과 같다. 예를 들어 다음과 같은 코드를 작성해 보자

```c++
// hw1과 chw1은 homework의 동의어
// chw1은 조회만 가능
vector<double>&hw1 = hw;
const vector<double>& chw = chw;
```

hw1은 hw와 마찬가지로 homework에 또 다른 이름이고 chw1은 chw처럼 조회만 할 수 있는 homework에 또 다른 이름이다.

const가 아닌 참조를 정의할 때는 const객체나 const 참조를 참조할 수 없다. const가 허용하지 않는 권한을 요청하는 것이기 때문이다.

즉 chw를 수정하지 않기로 약속했기 때문에 다음처럼 작성할 수 없다.

```c++
vector<double>& hw2 = chw; //오류: chw에 쓰기 권한을 요청
```

매개변수그 const vector<double>& 타입이면 인수를 복사하지 않고 인수에 접근할 수 있도록 요청하고 매개변수의 값을 바꾸지 않는다.

매개변수가 const 참조이므로 const 벡터와 const가 아닌 벡터 모두를 상대로 함수를 호출할 수 있고, 매개변수가 참조 형태이므로 인수가 복사할 때의 오버헤드를 피할 수 있다.



두번째 승미로운 점은 grade 함수이다. grade 안에서 grade 함수를 호출한다. 이름이 같은 함수들을 사용하는 개념을 **오버로딩**이라고 한다.

비록 두 함수의 이름이 같지만 grade 함수를 호출할 때마다 제공하는 인수 목록에서 세 번째 인수의 타입이 다르므로 서로 다른 함수라는 것을 구분한다.



세번째 흥미로운 점은 median 함수와 별개로 homework.size()가 0인디 확인한다.

median 함수에서 중앙값을 구하려는 벡터가 비어 있으면 'median of an empty vector'라는 메세지와 함께 예외를 던지는데 학생 성적을 구하려는 사람에게 이는 직접적인 도움이 되지 않는다. 따라서 상황에 맞는 예외를 던저 사용자에게 무엇이 잘못되었는지 더 많은 단서를 줄 수 있다.



### 4.1.3 read_hw 함수

벡터에 과제 넣는 부분을 함수로 만들자. 그러면 한 번에 값 2개를 반환해야 한다. 2개의 값은 입력된 과제 점수에 해당하는 값과 입력이 제대로 일어났는지 나타내는 값이다.

함수가 직접 하나 이상의 값을 반환하는 방법은 없다. 간접적으로 값 2개 중 하나를 담은 객체의 참조를 함수의 매개변수로 지정하는 것이다.

이 방식은 일반적으로 입력을 받은 함수에 사용한다.

```c++
// 입력스트림에서 과제 점수를 읽어서, vector<double>에 넣음
istream& read_hw(istream& in, vector<double>& hw)
{
	// 앞으로 완성시켜야 할 부분
	return in;
}
```

다음 코드를 보자

```c++
vector<double> homework;
read_hw(cin, homework);
```

read_hw함수의 두 번째 매개변수가 참조 타입일 때는 read_hw 함수를 호출하면 homework의 값이 변경될 수 있음을 예상해야 한다.

함수에서 인수를 바꿀 것이 예상되므로 표현식 형태의 인수로 함수를 호출할 수 없다. 대신 lvalue 인수를 참조 매개변수로 전달한다.

lvalue라는 것은 **비일시적 객체**를 나타내는 값이다.

예를 들어 참조인 변수 혹은 참조를 반환하는 호출했을 때 결과인 변수가 lvalue이다. sum / count 처럼 산술 값을 만드는 표현식은 lvalue가 아니다.



read_hw 함수의 두 매개변수는 참조이므로 함수가 두 인수의 상태를 바꿀 것이라 예상한다.

cin은 라이브러리가 입려 파일의 상태에 관해 알아야 할 모든 것을 저장하는 데이터 구조인 것으로 알고 있다.

표준 입력 파일에서 입력된 값을 읽으면 파일의 상태가 바뀌므로 논리적으로 생각했을 때 cin값도 변해야 한다.

read_hw 함수는 참조 타입인 in을 반환한다. 결론적으로 객체를 복사하지 않고 받은 후 , 복사하지 않을 채로 반환할 것이다. 입력 스트림을 반환함으로 다음처럼 코드를 작성할 수 있다/

```c++
if (read_hw(cin, homework)){ */ 생략 */ }
```

앞 코드는 다음과 같은 의미이다.

```c++
read_hw(cin, homework);
if (cin) { */ 생략 */ }
```



과제 입력받는 방법을 생각해 보자. 모든 과제 점수를 입력받아야 하므로 일단 다음 형태로 코드를 작성하자.

일단 다음과 같이 코드를 작성하자.

```c++
double x;
while (in >> x)
    hw.push_back(x);
```

위의 코드는 제대로 동작하지 않는다. 첫 번째로는 hw를 정의하지 않았다. 이미 hw 안에 이미 어떤 데이터가 있을지도 모르기 때문에 입력받은 작업을 시작하기 전 hw.clear()를 호출하여 이 문제를 해결할수 있다.



두번째로는 언제 반복문이 멈출 지 알 수 없다. 

```c++
while (in >> x)
```

이 반복문은 입력 실패하면 멈추는데 실패 이유는 여러가지이다.

- 경우 1 : 진짜 끝 -> 더이상 읽을 데이터가 없음 ex) 10 20 30
- 경우 2: 숫자가 아닌 걸 만남 ex) 10 20 30 x

여기서 문제는 반복문이 멈췄다고 해서 '경우 1'인지 '경우 2'인지 알 수 없다. 즉 멈춘 이유가 EOF일 수도 있고 잘못된 입력일 수도 있다.

in >> x가 실패하면 스트림은 **실패 상태**가 되며. 실패상태는 EOF에 도달한 것과 같은 의미로 , 그 다음부터는 입력이 안된다. 예를 들면

```c++
double x;
cin >> x;   // 여기서 실패
cin >> x;   // 이것도 계속 실패
cin >> x;   // 이것도 실패
```

그래서 in.clear를 사용한다. 이건

**“실패 상태를 지워라. 다시 입력 시도할 수 있게 해라.”** 라는 뜻이다.

중요한건 clear()는 **"오류 상태만 지운다"** 이상한 문자 자체를 지우지 않는다.

이렇게 남아있으면

```c++
10 20 x 30
```

`x`에서 실패한 뒤 `in.clear()`만 하면, 다시 읽으려고 할 때 또 `x`를 만나서 또 실패할 수 있다.

하지만 점수가 아닌 값을 만나면 읽지 않은 채로 남겨두는 것이 좋은데 

```c++
100 90 80 done
```

이고 done에 실패하더라도 입력 스트림 안에 남아있고 나중에 호출하는 쪽이 done을 다른 용도로 읽을 수 있기 때문이다.



마지막으로 첫번째 과제 점수를 일기 전에 이미 입력이 끝났을 수도 있다. 예를 들어 아예 입력이 없을 수도 있고 첫 글자가 숫자가 아닐 수도 있다.

이 경우에는 첫 번째 in >> x부터 실패하고 그러면 hw에는 아무 점수도 안들어가기 때문에 호출자에게 아무 점수도 안들어가기 때문에 스트림을 다시 쓸 수 있게 clear()를 해줘야 한다.

read_hw함수를 만들어 보자

```c++
// 입력스트림에서 과제 점수를 읽어서, vector<double>에 넣음
istream& read_hw(istream& in, vector<double>& hw)
{
	if (in) {
		// 이전 내용을 제거
		hw.clear();

		// 과제 점수를 읽음
		double x;
		while (in >> x)
			hw.push_back(x);

		// 다음 학생 입력을 위해 스트림을 초기화
		in.clear();
	}
	return in;
}
```



### 4.1.4 함수의 매개변수

지금까지 homework를 다루는 3개 함수(median, grade, read_hw) 함수를 정의했다. 각 함수는 목적에 맞게 서로 다른 매개변수를 처리한다.

함수 median(4.1.1)의 매개변수는 vector<double> 타입이므로 함수를 호출하면 크기에 상관없이 복사한다. 비효율 적이지만 벡터의 중앙값을 구할 때 벡터가 바뀌는 것을 피할 수 있으므로 median의 매개변수로 적합하다.

homework(4.1.2)를 다루는 grade함수의 매개변수는 const vector<double>& 타입이다. 여기서 &는 인수를 복사하지 않도록 구현체에 요청하ㅏ는 역할이고 const는 프로그램이 매개변수를 바꾸지 않도록 한다. 매개변수가 벡터나 문자열 처럼 복사하는데 시간이 필요하거나 매개변수의 값을 바꾸지 않으려 할 때문 const vector<double>& 타입을 사용하는 것이 적합하다.

read_hw의 매개변수는 vector<double>& 타입이다. 함수에서 인수의 값을 바꾸기 때문에 const가 아니다.

const가 아닌 참조 매개변수에 해당하는 인수는 반드시 lvalue 즉, 비일시적 객체여야 한다. 반면 값으로 전달되거나 const 참조 타입인 인수는 어떠한 값도 될 수 있다. 

예를 들어 빈 벡터를 반환하는 함수가 있다고 가정해보자

```c++
vector<double> emptyvec()
{
    vector<double> v;
    return v;
}
```

이 함수를 호출하고 그 결과를 grade 인수에 사용하면

```c++
grade(midterm,final,emptyvec());
```

실행하면 grade는 인수가 비었으므로 즉시 예외를 던진다.



read_hw를 호출할때는 두 매개변수가 const가 아닌 참조 타입이기에 두 인수 모두 lvalue여야 한다. 

```c++
read_hw(cin,emptyvec()) // 오류 emptyvec()는 lvalue가 아님
```

함수 emptyvec을 호출하여 만드는 벡터는 함수 read_hw의 실행이 끝나면 사라질 것이므로 컴파일러는 오류를 남긴다. 즉, 접근할 수 없는 객체에 입력을 저장하는 것과 같다.



### 4.1.5 함수를 사용하여 학생 성적 구하기

```c++
#include <iostream>
#include <iomanip>
#include <vector>
#include <algorithm>
#include <stdexcept>
#include <string>
#include <ios>
using std::cout; using std::setprecision;
using std::vector; using std::sort;
using std::domain_error; using std::istream;
using std::string; using std::cin; using std::endl;
using std::streamsize;

double median(vector<double> vec)
{
	typedef vector<double>::size_type vec_sz;
	vec_sz size = vec.size();
	if (size == 0)
		throw domain_error("median of an empty vector");
	sort(vec.begin(), vec.end());
	vec_sz mid = size / 2;
	return size % 2 == 0 ? (vec[mid] + vec[mid - 1]) / 2 : vec[mid];
}

double grade(double midterm, double final, double homework)
{
	return 0.2 * midterm + 0.4 * final + 0.4 * homework;
}

double grade(double midterm, double final, const vector<double>& hw)
{
	if (hw.size() == 0)
		throw domain_error("student has done no homework");

	return grade(midterm, final, median(hw));
}



// 입력스트림에서 과제 점수를 읽어서, vector<double>에 넣음
istream& read_hw(istream& in, vector<double>& hw)
{
	if (in) {
		// 이전 내용을 제거
		hw.clear();

		// 과제 점수를 읽음
		double x;
		while (in >> x)
			hw.push_back(x);

		// 다음 학생 입력을 위해 스트림을 초기화
		in.clear();
	}
	return in;
}

int main()
{
	// 학생의 이름을 입력받음
	cout << "Please Enter your First name: ";
	string name;
	cin >> name;
	cout << "Hello, " << name << "!" << endl;

	// 학생의 중간고사와 기말고사 점수를 입력받음
	cout << "Please Enter your midterm and final exam grades: ";
	double midterm, final;
	cin >> midterm >> final;

	// 과제 점수를 읽어서 vector<double>에 넣음
	cout << "Enter all your homework grades, "
		"followed by end-of-file: ";
	vector<double> homework;

	read_hw(cin, homework);

	try {
		double final_grade = grade(midterm, final, homework);
		streamsize prec = cout.precision();
		cout << "Your final grade is " << setprecision(3)
			<< final_grade << setprecision(prec) << endl;
	}
	catch (domain_error) {
		cout << endl << "you must enter your grades. "
			"please try again." << endl;
		return 1;
	}
	return 0;
}
```

위 프로그램에서 새로운 개념은 try문이다. 중괄호 속 실행문을 실행하다 domain_error 예외가 발생하면 실행문의 실행을 멈추고 catch다음에 오는 중괄호 안 실행문을 실행한다.

예외를 만들지 않고 완료되면 catch문을 건너 뛰고 return 0;으로 넘어간다.



try문을 작성할때 의도와 다른 결과를 얻지 않도록 주의해야 한다 예를 들면

```c++
// 이 예제는 동작하지 않음
try{
    streamsize prec = cout.precision();
    cout << "Your final grade is " << setprecision(3)
    << grade(midterm, final, homework) << setprecision(prec) << endl;
}
```

이 코드는 문제를 일으킬 수 있다 grade 함수 실행시 에러가 나서 catch문으로 넘어가면 cout에 setprecision(3)으로 변경한 후 이전 값으로 초기화 하는 setprecision(prec)으로 재설정 하지 않는다.

이래서 완성코드는 출력 블록을 2개의 실행문으로 분리해서 작성하였다. 

따라서 예외를 일으킬 수 있는 실행문은 입력 및 출력을 비롯하여 부수적으로 일어나는 어떠한 결과를 만들어 내지 않아야 한다.



## 4.2 데이터 구조화

이 코드는 학생 1명의 성적을 구하는데 유용하지만 모든 학생에 성적을 구해야 하도록 프로그램을 수정할 수 있다ㅏ.

학생 점수를 대화 형태로 입력받은 대신 학생의 이름과 점수가 포함된 파일을 제공한다고 가정해보자.

각 이름 뒤에는 중간고사, 시험점수, 기말 점수, 하나 이상의 과제 점수가 나열된다.

```tex
Smith 93 91 47 90 92 73 100 87
Carpenter 75 90 87 92 93 60 0 98
...
```

입력에 따른 출력 형태는 다음과 같다.

```tex
Carpenter 86.8
Smith     90.4
```

화면에서 학생 이름을 알파벳 순으로 나열하고 최종 점수를 읽기 쉽게 수직으로 정렬하고자 한다.

이러면 모든 학생의 기록을 저장할 공간이 필요하고 각 이름과 해당 점수사이에 들어갈 공백의 개수를 결정해야 하므로 가장 긴 학생의 이름을 알아야 한다.



### 4.2.1 학생 데이터 통합하기

위의 기능을 구현하려면 이름과 점수를 한군데 모아야 하고, 학생 1명과 관련된 모든 정보를 하나의 공간안에 저장할 수 있도록 데이터 구조가 필요하다.

c++에서는 데이터 구조를 다음처럼 정의한다.

```c++
struct Student_info {
	string name;
	double midterm, final;
	vector<double> homework;
}; // 세미콜론 있어야 함
```

Student_info는 4개의 데이터 멤버가 있는 타입이다. 각 객체는 1명의 정보를 담고 Student_info는 타입이므로 vector<Student_info>의 객체를 사용할 수 있다.



### 4.2.2 학생 점수 관리

학생들의 점수를 다루려면 3단계의 개별 함수로 표현할 수 있다.

Student_info에 데이터 입력, 종합 점수 생성, Student_info객체들을 요소로 갖는 벡터를 정렬해야 한다.

Student_info 객체에 데이터를 입력하는 함수는 4.1.3에서 확인한 read_hw함수와 비슷하다. 또한 학생의 이름과 시험 점수도 읽어야 한다.

```c++
istream& read(istream& is, Student_info& s)
{
	is >> s.name >> s.midterm >> s.final;
	read_hw(is, s.homework);
	return is;
}
```

함수 이름이 read가 모소해 볼 수 있으나 두번째 매개변수와 타입을 보면 이 함수는 학생 정보라는 것을 확실히 알 수 있다.

2개의 참조 매개변수가 있다. 하나는 읽어야할 istream 그리고 저장할 객체이다. s는 참조 타입이므로 함수안에서 s를 사용해면 전달한 인수 상태의 영향을 준다.

read함수는 s의 멤버인 name, midterm, final의 값을 읽고 read_hw 함수를 호출하여 읽는다.

이 과정에서 EOF에 도달하거나 입력 오류가 발생하면 read함수로 돌아오고 is는 실패상태가 된다. 만약 read_hw함수를 호출할 때 is가 오류를 기록하고 있으면 read_hw함수는 입력 스트림을 오류 상태로 두고 그대로 내버려 둔다.



Student_info의 최종 점수를 구하는 함수를 만들어 보자

```c++
double grade(const Student_info& s)
{
	return grade(s.midterm, s.final, s.homework);
}
```

Student_info 타입을 사용해 double 타입의 종합 점수를 구해 반환한다.

참조 타입의 인수이기 때문에 객체를 통째로 복사하지 않아 발생하는 오버헤드를 피할 수 있다.

내부에서 호출하는 grade 함수가 일으키는 예외에 대응하지 않는다.

내부의 grade 함수는 이미 실행이 끝난 이후라면 예외를 다루기 위해 함수가 할 수 있는 것이 아무것도 없다. 

어떤 예외가 발생하더라도 이 함수를 호출한 지점으로 전달될 것이고 지금 다루는 함수에서는 과제를 하지 않은 학생을 대상으로 어떤 동작을 할 지 정하는데 신경쓸 필요가 없다.



마지막으로는  Student_info의 객체의 벡터를 어떻게 정렬할지 정하는 것이다.

```c++
sort(student.begin(), student.end()) // error!
```

위의 코드는 에러가 있다 sort 함수는 순서를 비교할 때 < 연산자를 사용하는데 Student_info 객체에는 사용할 수 없기 때문이다.

하지만 sort함수는 세 번째의 인수로 **서술 함수**가 있다.

서술 함수는 bool 진릿값을 반환하는 함수이며 sort 함수에 세 번째 인수가 있다면 < 대신 세번째 인수인 함수를 사용하여 요소들을 비교한다.

따라서 Student_info 타입인 인수 2개가 있는 함수를 정의하여 첫 번째 인수가 두 번째 인수보다 작은지 판별해야 한다.

학생 이름을 알파벳순으로 정렬하려면 다음처럼 이름만 비교하는 함수를 만들어야 한다.

```c++
bool compare(const Student_info& x, const Student_info& y)
{
	return x.name < y.name;
}
```

이 함수는 문자열을 비교하려고 < 연산자를 제공하는 문자열 클래스에 Student_info를 비교하는 작업을 맡긴다. 여기서 < 연산자는 일반적인 사전 순서대로 문자열을 비교한다.

이제 sort 함수의 세번째 인수에 compare함수를 넣으면 벡터를 정렬할 수 있다.

```c++
sort(students.begin(), students.end(), compare);
```



### 4.3.3 점수 보고서 만들기

이제 학생 점수를 다루는 함수를 구현했으므로 점수 보고서를 만들 수 있다.

```c++
#include <iostream>
#include <iomanip>
#include <vector>
#include <algorithm>
#include <stdexcept>
#include <string>
#include <ios>

using std::cout; using std::setprecision;
using std::vector; using std::sort;
using std::domain_error; using std::istream;
using std::string; using std::cin; using std::endl;
using std::streamsize; using std::max;

struct Student_info {
	string name;
	double midterm, final;
	vector<double> homework;
}; // 세미콜론 있어야 함

bool compare(const Student_info& x, const Student_info& y)
{
	return x.name < y.name;
}

// 입력스트림에서 과제 점수를 읽어서, vector<double>에 넣음
istream& read_hw(istream& in, vector<double>& hw)
{
	if (in) {
		// 이전 내용을 제거
		hw.clear();

		// 과제 점수를 읽음
		double x;
		while (in >> x)
			hw.push_back(x);

		// 다음 학생 입력을 위해 스트림을 초기화
		in.clear();
	}
	return in;
}

istream& read(istream& is, Student_info& s)
{
	is >> s.name >> s.midterm >> s.final;
	read_hw(is, s.homework);
	return is;
}

double median(vector<double> vec)
{
	typedef vector<double>::size_type vec_sz;
	vec_sz size = vec.size();
	if (size == 0)
		throw domain_error("median of an empty vector");
	sort(vec.begin(), vec.end());
	vec_sz mid = size / 2;
	return size % 2 == 0 ? (vec[mid] + vec[mid - 1]) / 2 : vec[mid];
}

double grade(double midterm, double final, double homework)
{
	return 0.2 * midterm + 0.4 * final + 0.4 * homework;
}

double grade(double midterm, double final, const vector<double>& hw)
{
	if (hw.size() == 0)
		throw domain_error("student has done no homework");

	return grade(midterm, final, median(hw));
}

double grade(const Student_info& s)
{
	return grade(s.midterm, s.final, s.homework);
}

int main()
{
	vector<Student_info> students;
	Student_info record;
	string::size_type maxlen = 0; // 학생 이름의 최대 길이


	// 학생의 이름과 모든 점수를 읽어 저장하고 가장 긴 이름을 찾음
	while (read(cin, record)) {
		maxlen = max(maxlen, record.name.size());
		students.push_back(record);
	}

	// 학생들을 이름순으로 정렬
	sort(students.begin(), students.end(), compare);

	for (vector<Student_info>::size_type i = 0; i < students.size(); ++i) {
		cout << students[i].name << string(maxlen + 1 - students[i].name.size(), ' '); // 이름과 점수 사이에 공백 추가
		try {
			double final_grade = grade(students[i]);
			streamsize prec = cout.precision();
			cout << setprecision(3) << final_grade << setprecision(prec);
		} catch (domain_error e) {
			cout << e.what();
		}
		cout << endl;
	}
}
```

max 함수의 호출을 살펴보자. max함수의 두 인수는 같은 타입이어야 한다. 그리고 int타입이 아닌 string::size_type으로 정의해야한다.



다음으로

```c++
string(maxlen + 1 - students[i].name.size(), ' ')
```

이 표현식은 문자열 타입에 이름없는 객체를 만든다. 이 객체는 maxlen + 1 - students[i].name.size()개의 공백문자가 있다.

이 표현식은 변수의 이름이 없는데 변수 이름을 생략하면서 표현식으로 객체를 정의할 수 있다.



for문에서는 i를 인덱스로 사용해서 모든 students의 요소를 거쳐  name멤버에 알맞게 구성된 빈 문자열을 덧붙여 출력한다.



다음은 각 학생의 최종 점수를 출력한다. 학생이 숙제를 하지 않았다면 최종 점수의 계산 과정에서 예외가 발생한다.

이때 catch문에서 예외를 처리하며 최종 점수를 출력하는 대신 예외 객체의 일부를 구성하는 메세지를 출력한다.

domain_error 처럼 표준 라이블리가 정의한 모든 예외는 예외가 발생할 이유을 설명할 때 선택해서 사용할 수 있는데 what이라는 멤버함수로 해당 내용을 복사할 수 있다.

graade 함수(4.1.2)에서 발생하는 예외 객체에 e라는 이름을 부여하므로 what 함수를 호출해서 얻은 메세지를 출력할 수 있다.



## 4.2 프로그램 통합하기

지금까지 작성한 프로그램들은 매우 복잡해졌다. C++에서는 이런 복잡성을 줄이려고 **개별컴파일**에 개념을 지원한다. 

프로그램을 개별파일에 저장하고 각 파일을 독립적으로 컴파일할 수 있다.



median함수를 패키지로 만들어 보자. median함수의 정의문을 하나의 파일에 저장하여 별도로 컴파일할 수 있다.

이 파일은 median 함수가 사용하는 모든 이름의 선언을 반드시 포함해야 한다. median 함수는 라이브러리부터 벡터 타입, sort함수, domain_error 예외를 가져와 사용하므로 이 기능에 적합한 헤더를 포함해야 한다.

```c++
// median 함수의 소스파일
#include <algorithm>
#include <stdexcept>
#include <vector>

using std::vector; 
using std::sort;
using std::domain_error;

double median(vector<double> vec)
{
	typedef vector<double>::size_type vec_sz;
	vec_sz size = vec.size();
	if (size == 0)
		throw domain_error("median of an empty vector");
	sort(vec.begin(), vec.end());
	vec_sz mid = size / 2;
	return size % 2 == 0 ? (vec[mid] + vec[mid - 1]) / 2 : vec[mid];
}

```

다음 단계는 median 함수를 다른 프로그래머가 사용할 수 있게 만드는 것이다.

표준 라이브러리가 정의한 이름은 헤더에 저장되는데 이와 같은 원리로 다른 프로그래머가 우리가 정의한 함수를 이름으로 사용할 수 있도록 직접 **헤더 파일**을 만들 수 있다.

헤더 파일에 median 함수가 있다면 다음과 같은 방법으로 사용할 수 있다.

```c++
#include "median.h"
#include <vector>
```

#include 지시문을 사용할 때 큰따옴표로 헤더이름을 둘러싸는 것은 #include 지시문 자리에 해당 헤더 파일 내용 전체를 복사하라는 의미다.

헤더 파일안에는 median 함수의 **선언문**이 있어야 한다. 함수를 선언하는 방법은 본문 대신 그 자리에 세미콜론을 작성하는 것이다. 또한 함수 본문과 관련이 없기에 매개변수의 이름을 없앨 수 있다.

```c++
double median(vector<double>);
```

헤더 안에는 이러한 선언문과 선언문에서 사용하는 이름도 포함해야 한다. 선언문에서 벡터를 사용하므로 컴파일러가 선언문을 확인하기 전 해당 이름을 사용해야 한다.



```c++
// median.h
#include <vector>

double median(std::vector<double> vec);
```

vector의 헤더를 포함하면 median 함수의 인수를 선언할 때 std::vector를 사용할 수 있는데 이 때 using 선언문을 하면 안된다.

**일반적으로 헤더파일에는 필요한 이름의 선언해야 한다.**

using std::vector를 사용하게 되면 해당 헤더를 포함하는 모든 프로그램은 원하든 원치 않던 std::vector에 using 선언이 있기 때문이다.



마지막으로 헤더파일은 프로그램을 컴파일 할 때 두 번이상 포함되면 안된다. 그래서 모든 파일에 헤더파일에 대한 전처리기를 해줘야 한다.

```c++
#ifndef GUARD_MEDIAN_H
#define GUARD_MEDIAN_H

#include <vector>

double median(std::vector<double> vec);

#endif
```

#ifndef 지시문은 GUARD_MEDIAN_H가 정의되었는지 확인한다. 이를 **전처리기 변수**라고 한다.

#ifndef 지시문은 주어진 이름이 정의되지 않으면 #endif를 만나기 전까지 모든 것을 처리하도록 전처리기에 요청한다.

프로그렘에 median.h가 포함되었다면 GUARD_MEDIAN_H가 정의되지 않았으므로 파일의 나머지 부분을 처리한다.

먼저 GUARD_MEDIAN_를 정의한다, 그러면 이후에 median.h를 포함하더라도 중복된 결과가 나오지 않는다.



## 4.4 성적 산출 프로그램 분할하기

이제 Student_info 구조체와 관련된 함수를 패키지로 만들어 보자.

```c++
#ifndef GUARD_STRING_INFO_H
#define GUARD_STRING_INFO_H

#include <iostream>
#include <string>
#include <vector>

	std::string name;
	double midterm, final;
	std::vector<double> homework;
}; // 세미콜론 있어야 함

bool compare(const Student_info&, const Student_info&);

```

```c++
// Student_info 구조체와 관련된 함수들의 소스파일
#include "Student_info.h"

using std::vector; using std::istream;

bool compare(const Student_info& x, const Student_info& y)
{
	return x.name < y.name;
}

istream& read(istream& is, Student_info& s)
{
	is >> s.name >> s.midterm >> s.final;
	read_hw(is, s.homework);
	return is;
}

istream& read_hw(istream& in, vector<double>& hw)
{
	if (in) {
		hw.clear();

		double x;
		while (in >> x)
			hw.push_back(x);

		in.clear();
	}
	return in;
}

```

이렇게 분리해서 작성할 때는 함서의 선언과 정의를 검사할 때 **반환 타입**, **매개변수의 개수 및 타입**이 완벽하게 일치해야 한다.

선언과 정의 부분이 다르면 구현체는 함수가 두 가지 다른 버전으로 오버로드되어있다고 판단한다.



grade함수를 선언하는 헤더파일을 만들어 보자.

```c++
#ifndef GUARD_STRING_INFO_H
#define GUARD_STRING_INFO_H
// grade.h
#include <vector>
#include "Student_info.h"

double grade(double, double, double);
double grade(double, double, const std::vector<double>&);
double grade(const Student_info&);

#endif
```

코드처럼 오버로드 된 함수를 한군데에 선언하면 관계를 더 쉽게 확인할 수 있어서 좋다.

```c++
// grade.cpp
#include <stdexcept>
#include <vector>
#include "median.h"
#include "grade.h"
#include "Student_info.h"

using std::vector; using std::domain_error;

double grade(double midterm, double final, double homework)
{
	return 0.2 * midterm + 0.4 * final + 0.4 * homework;
}

double grade(double midterm, double final, const vector<double>& hw)
{
	if (hw.size() == 0)
		throw domain_error("student has done no homework");

	return grade(midterm, final, median(hw));
}

double grade(const Student_info& s)
{
	return grade(s.midterm, s.final, s.homework);
}
```



## 4.5 수정된 성적 산출 프로그램

```c++
#include <algorithm>
#include <iomanip>
#include <ios>
#include <iostream>
#include <stdexcept>
#include <vector>
#include <string>
#include "grade.h"
#include "Student_info.h"

using std::cin; using std::setprecision;
using std::cout;  using std::sort;
using std::domain_error; using std::streamsize;
using std::string; using std::endl;
using std::max; using std::vector;

int main()
{
	vector<Student_info> students;
	Student_info record;
	string::size_type maxlen = 0; // 학생 이름의 최대 길이


	// 학생의 이름과 모든 점수를 읽어 저장하고 가장 긴 이름을 찾음
	while (read(cin, record)) {
		maxlen = max(maxlen, record.name.size());
		students.push_back(record);
	}

	// 학생들을 이름순으로 정렬
	sort(students.begin(), students.end(), compare);

	for (vector<Student_info>::size_type i = 0; i < students.size(); ++i) {
		cout << students[i].name << string(maxlen + 1 - students[i].name.size(), ' '); // 이름과 점수 사이에 공백 추가
		try {
			double final_grade = grade(students[i]);
			streamsize prec = cout.precision();
			cout << setprecision(3) << final_grade << setprecision(prec);
		} catch (domain_error e) {
			cout << e.what();
		}
		cout << endl;
	}
	return 0;
}
```



## 4.6 핵심 정리

**프로그램 구조**

프로그램 구조를 만드는데 사용하는 #include 지시문의 사용 방법은 다음과 같다

- #include <시스템-헤더> : 꺽쇠괄호 <>는 '시스템 헤더'를 묶는다. "시스템 헤더"는 파일로 구현될 수도 있고 아닐 수도 있다.
- #include "사용자 정의 파일 이름" : 사용자 정의 헤더 파일은 이름을 큰따옴포료 묶는다.

헤더 파일은 #ifndef GUARD_헤더이름 이라는 지시문을 파일로 묶어 중복사용을 방지한다.

헤더 내부에는 사용하지 않는 이름을 선언할 수 없다. 특히 using 선언문을 포함하면 안된다.



**타입**

- T& : T 타입의 참조를 나타낸다. 함수 내부에서 바뀌는 매개변수를 전달하는데 사용하며 이러한 매개변수의 인수는 lvalue이어야 한다.
- const T& : T타입의 참조를 나타내며 바인딩 된 값을 변경하는데 사용할 수 없다. 보통 매개변수를 함수로 복사하는  비용을 피하는데 사용된다



**구조체**

구조체는 0개 이상의 멤버를 갖는 타입이다. 구조체 타입의 객체마다 멤버별 인스턴스가 있고 모든 구조체는 정의 문이 있어야 한다.

```c++
struct 타입-이름{
	타입-지정자 멤버-이름;
	...
}; // 세미콜론 필수
```

모든 정의문과 마찬가지로 구조체 정의문은 소스 파일 당 한 번만 등장하면 되고, 일반적으로 잘 보호된 헤더 파일에 존재한다.



**함수**

함수는 해당 함수를 사용하는 모든 소스파일에서 선언해야 하며 한 번만 정의해야 한다.

```
반환-타입 함수-이름 (매개변수-선언) // 함수 선언문
[inline] 반환-타입 함수-이름 (매개변수-선언){ // 함수 정의문
	// 함수 본문
}
```

함수는 호출하기 전 반드시 선언해야 한다. 

인수 각각의 타입은 상응하는 매개변수와 호환할 수 있어야 한다.

함수 이름은 오버로드 될 수 있다. 같은 함수 이름으로 매개 변수 개수 또는 타입이 서로 다른 여러 함수를 정의할 수 있다. 같은 타입의 참조와 const 참조를 구현할 수 있다.

필요하다면 함수 정의에 inline을 붙일 수 있으며 상황에 따라 함수 호출을 인라인으로 확장하도록 컴파일러에게 요청한다.

즉 함수 호출의 오버헤드를 피하려고 함수 호출 각각을 함수 본문의 복사본을 치환한다. 이 때 컴파일러가 함수의 정의문을 파악하고 있어야 하므로 인라인 함수는 일반적으로 소스파일이 아닌 헤더파일 내부에 정의한다.



**예외처리**

예외 처리와 관련된 구문은 다음과 같다.

- try{ // 코드 : 예외를 던질 수 있는 코드 블록을 시작한다
- } catch(t) {/* 코드 */} : try 실행문을 끝내고 t타입에 해당하는 예외를 처리한다. catch문 다음 코드에서는 t에 해당하는 예외를 처리할 모든 동작을 실행한다.
- throw e; : e 값을 반환하여 현재 함수를 종료한다.



**예외 클래스**

라이브러리는 몇가지 예외 클래스를 정의한다. 각 예외 클래스의 이름으로부터 발생한 문제의 종류를 추측할 수 있다.

- logic_error
- domain_error
- invalid_argument
- length_error
- out_of_range
- runtime_error
- range_error
- overflow_error
- underflow_error

참고로 e.what()은 오류를 일으킨 상황을 알려주는 값을 반환한다.



**라이브러리 기능**

라이브러리 기능에는 다음과 같이 있다.

- s1 < s2 : 문자열 s1 과 s2를 사전 순서대로 비교한다.
- s.width(n) : 다음 출력 동작을 위해 스트림 s의 너비를 n으로 설정한다
- setw(n) : 출력 스트림 s의 내용을 출력할 때 s.width(n)을 호출한 효과가 있는 streamsize 타입 값을 반환한다.
