# 5. 순차 컨테이너와 문자열 분석

표준 라이브러리는 유용한 데이터와 함수를 제공할 뿐만 아니라 일관된 아키텍처를 반영한다.

따라서 한 종류의 컨테이너가 어떻게 동작하는지 학습했다면 모든 라이브러리 컨테이너를 다루는 방법을 이해할 수 있다.



## 5.1 학생 분류

4.2에서 다룬 성적 산출 문제를 다시 논의해보자. 모든 학생의 최종 점수를 계산하는 것 뿐만 아니라 어떤 학생들이 과락 대상인지도 알아볼 수 있다.

학생 정보가 있는 벡터에서 어떤 과목의 기준 점수를 통과하지 못한 학생들의 정보를 추출하여 다른 벡터에 저장하고, 원래의 벡터에서 과락한 학생들의 데이터를 제거하여 해당 과목의 기준 점수를 통과한 학생들의 정보만 남겨 보자

우선 학생의 과락 여부를 판별하는 간단한 함수를 만들어 보자.

```c++
// 학생의 과락 여부를 결정하는 서술 함수
bool fgrade(const Student_info& s)
{
	return grade(s) < 60;
}
```



주어진 문제를 가장 직접적으로 해결하는 방법은 해당 과목의 기준점수를 통과한 학생과 통과하지 못한 학생의 벡터를 각각 만들고 학생 각각의 정보를 조사하여 두 벡터 중 하나에 넣는 것이다.

```c++
vector<Student_info> extract_fails(vector<Student_info>& students)
{
	vector<Student_info> pass, fails;

	for (vector<Student_info>::size_type i = 0; i < students.size(); ++i) {
		if (fgrade(students[i]))
			fails.push_back(students[i]);
		else
			pass.push_back(students[i]);
	}
	students = pass;
	return fails;
}
```

이 함수는 실제로 2개의 벡터를 만든다. 

하나는 extract_fails 함수가 봔환하는 과락 학생의 벡터, 하나는 함수를 호출할 때 생성하는 기준 점수 통과 학생의 벡터이다.

students는 참조 타입이므로 매개변수를 바꾸면 인수에 반영된다.



### 5.1.1 요소 제거

extract_fails 함수는 원하는 동작을 수행하지만 단점이 있다. pass벡터와 fail 벡터를 만들었어도 원본 벡터는 여전히 남아있으므로 학생의 정보를 2개씩 보관하려면 충분한 메모리 공간이 필요하다.

필요 없는 여러개에 데이터 복사본을 유지하지 않도록 만들자.

한가지 방법은 pass 벡터를 없애는 것, students 벡터에 있는 학생의 최종 점수를 계산하여 기준 점수를 통과한 학생의 정보만을 남기고 과락 학생의 정보는 복사본 fail 벡터에 추가한 후 students 벡터에서 제거할 것이다.

이런 방식은 벡터의 요소를 제거하는 기능이 있어야 한다.

단, 벡터에서 요소를 제거하는 작업은 대용량의 입력데이터를 처리할 때 속도가 느리다.

벡너는 빠르게 데이터 임의로 접근하는데 최적화 되어 있지만 벡터의 끝 이외 위치의 요소를 삽입하거나 제거할 때 성능이 나빠질 수 있다.

성능 문제를 해결하는 방법은 2가지 방법이 있다.

- 더 적합한 데이터 구조를 사용하는 것
- 오버헤드를 피하는 더 똑똑한 알고리즘을 사용하는 것

성능이 좋지 않은 코드를 먼저 살펴 보자

```c++
vector<Student_info> extract_fails(vector<Student_info>& students)
{
	vector<Student_info> fails;
	vector<Student_info>::size_type i = 0;
	while (i != students.size()) {
		if (fgrade(students[i])) {
			fails.push_back(students[i]);
			students.erase(students.begin() + i);
		}
		else
			++i;
	}
	return fails;
}
```

새롭게 살펴볼 것은 벡터에서 요소를 제거하는 방법이다.

```c++
students.erase(students.begin() + i);
```

벡터 타입은 요소를 제거하는 역할인 erase라는 멤버함수가 있다. 인수는 제거하는 요소를 가리킨다.

student[i].erase()같은 형태의 함수는 존재하지 않는다. 모든 컨테이너가 인덱스를 지원하지 않기 때문이다.

따라서 라이브러리가 모든 컨테이너에서 같은 방식으로 동작하는 형태의 erase 함수를 제공하는 것이 더 유용하다.

students.begin()이 반환한 값에 인덱스를 더하여 제거할 요소를 가리킬 수 있다.

students.begin()는 벡터의 첫 번째 요소(인덱스가 0인 요소)를 반환한다. 그 값에 i를 더한 결과가 인덱스가 i인 요소를 가리킨다.

erase 함수는 벡터의 크기를 바꿀 뿐만 아니라 i의 요소를 제거하여 i가 순차열의 다음 요소를 가리키게 한다.

i 이후의 모든 요소는 하나 앞에 인덱스로 각각 복사된다. 

따라서 i를 바꾸지 않아도 다음 요소를 가르키기 때문에 i를 증가시키면 안된다.

만약 벡터의 요소가 과락 대상이 아니면 해당 요소를 students 벡터에 남겨야 하며 이때는 i를 증가시켜 다음 요소를 참조하도록 만들어야 한다.

students에 모든 요소를 확인했는지 알려면 i와 students.size()를 비교하면 되는데 하나를 제거하면 이전 보다 하나 더 적은 요소를 갖는다. 따라서 반복처리에 앞서 조건문으로 students.size() 함수를 호출해야 한다.

```c++
// 잘못된 최적화로 인해 동작하지 않음
vector<Student_info> size_type size = students.size();

while (i != size) {
	if (fgrade(students[i])) {
		fail.push_back(students[i]);
		students.erase(students.begin() + i);
	}
	else
		++i;
}

```

erase 함수를 호출하면 students 벡터에 있는 요소 개수가 변하므로 앞 코드는 제대로 동작하지 않는다.



### 5.1.2 순차적 접근 vs 임의적 접근

앞서 살펴본 두 가지 버전의 extract_fails 함수는 한가지 공통점이 있는데 컨테이너 요소에 순차적으로 접근한다.

```
학생 0 → 학생 1 → 학생 2 → 학생 3 → ...
```

이런 식으로, extract_fails 함수는 학생 각각의 정보를 순서대로 탐색하여 처리한다.

근데 기존 코드는 

```c++
students[i];
```

이렇게만 보면 구현체는 순차저긍로 동작하지만, 코드로 보면 그렇게 보이지 않는다.

이유는 

- i는 마음대로 바꿀 수 있음
- 점프 가능
- 뒤로 갈 수도 있음

즉 코드만 보면 이게 순차 탐색인지 확실할 수 없다.

이유은 students 벡터 요소에 접근하는데 students[i]로 표기하는 것은 순차적이 아닌 어떠한 순서로든 students 벡터 요소에 접근할 수 있음을 암묵

적으로 밝힌 셈이다.

컨테이너 요소 접근에 접근하는 순서를 주목해야 하는 이유는 다음과 같다.

- 서로 다른 타입의 컨테이너는 성능도 서로 다르다
- 서로 다른 연산을 지원한다.

프로그램이 사용하려는 어떤 연산이 특정 타입의 컨테이너에서만 효율적이라면 해당 컨테이너를 사용하여 프로그램의 성능을 더 향상시킬 수 있다.

다시 말해 extract_fails 함수는 단지 순차적 접근만 필요로 하므로 임의에 요소에 접근하는 기능이 있는 인덱스를 사용할 필요가 없다.

대신 순차적 접근만을 지원하는 연산으로 컨테이너의 요소에 제한적으로 접근하도록 extract_fails 함수를 다시 작성할 것이다.

이를 위해 c++ 라이브러리는 라이브러리가 제어할 수 있는 방식으로 데이터 구조에 접근할 수 있게끔 반복자라는 타입을 제공한다.

라이브러리는 반복자를 이용해 프로그램이 효율적으로 구현되도록 돕는다.



## 5.2 반복자

인덱스 i를 사용한 방법은 students 벡터의 i 요소를 가져와서 fgrade 함수에 전달한다. 

i를 인덱스로만 사용하여 students 벡터 요소에 순차적으로 접근하며 i로 실행하는 연산은 i와 벡터의 크기를 비교할 떄와 i를 증가키실 때 뿐이다.

하지만 유감스럽게도 라이브러리가 이러한 의도를 알 방법은 없다. 이럴 때는 인덱스 대신에 반복자를 사용해 라이브러리가 상황을 알게 할 수 있다. **반복자**는 다음과 같은 역할을 한다.

즉 지금 코드는 결과적으로 순자척인 접근을 하지만 라이브러리 입장에서는 

- `students[i]` → "임의 접근 가능 (random access)"
- `i` 값 → 마음대로 바뀔 수 있음

즉 라이브러리는 이게 순차적으로 도는 건지, 중간 건너 뛰는 건지 모른다.

c++에서는 컨테이너마다 특징이 다르다.

| 컨테이너 | 특징             |
| -------- | ---------------- |
| vector   | 랜덤 접근 가능   |
| list     | 순차 접근만 가능 |

그런데 인덱스 방식은 vector에서만 자연스럽게 동작한다.



반복자는 다음과 같은 역할을 한다.

- 컨테이너와 컨테이너 내부 요소를 식별
- 해당 요소에 저장된 값 검사
- 컨테이너 내부 요소 사이의 이동 연산 제공
- 컨테이너를 효율적으로 사용하도록 연산을 제한



반복자를 사용하면 다음과 같이 수정할 수 있다.

```c++
for (vector<Student_info>::const_iterator iter = students.begin(); iter != students.end(); ++iter) {
	cout << (*iter).name << endl; // 이름과 점수 사이에 공백 추가
}
```



### 5.2.1 반복자 타입

벡터와 같은 모든 표준 컨테이너는 2개의 반복자 타입을 정의한다.

```c++
container-type::const_iterator
container-type::iterator
```

container-type 자리에는 vector<Student_info> 처럼 컨테이너 요소 타입을 포함하는 컨테이너 타입이 들어간다.

저장된 값을 바꾸려면 iterator를 값을 읽기만 하려면 const_iterator 타입을 사용한다.



```c++
vector<Student_info>::const_iterator iter = students.begin();
```

위 코드는 iter가 vector<Student_info>::const_iterator라는 타입임을 알려준다.

중요한 점은 iterator 타입에서 const_iterator 타입을 자동 변환할 수 있다.

students.begin()은 iterator 타입을 반환한다, 하지만 변수는 const iterator로 정의되었다.

iterator타입을 const_iterator로 바꿀 수 있지만 const_iterator타입을 iterator타입으로는 바꿀 수 없다. 즉, 단방향이다.



### 5.2.2 반복자 연산

begin함수와 end함수는 컨테이너에 관한 반복자 타입을 반환한다.

```c++
iter != stndents.end()
```

iter와 students.end()의 반환 값이 같으면 for문 실행이 종료된다.

++iter는 반복자를 증가시켜 students 벡터의 다음요소를 참조하도록 만든다. 이 증가 연산자는 반복자의 값을 컨테이너의 다음 요소를 바꾸는 역할이다.

for 본문에서의 iter는 출력하는 students 벡터 요소를 가르켜야 한다. 따라서 해당 요소에 접근하려고 **역참조 연산자** *를 호출한다.

연산자를 반복자에 사용하면 lvalue를 반환하고, lvalue는 반복자가 참조하는 요소이다. 따라서 다음과 같은 출력문으로 name멤버를 표준 출력에 넣을 수 있다

```c++
cout << (*iter).name << endl; 
```



### 5.2.3 문법적 편의

```

위의 코드는 다음과 같이 작성할 수 있다.

```



### 5.2.4 students.erase(students.begin() + i)

```c++
students.erase(students.begin() + i)
```

students.begin()의 반환 값은 students 벡터의 첫 번째 요소를 참조하는 반복자인 것과 begin()+i가 벡터의 i번째 요소를 참조하는 것이다.

즉, +연산자의 의미는 반복자와 인덱스의 타입에 따라 달라진다.



만일 students가 인덱스를 이용한 의미적인 접근을 지원하지 않는 컨테이너라면 students.begin()은 + 연산을 지원하지 않는 타입을 반환할 것이다. 이 때는 students.begin() + i는 컴파일 할 수 없다.

이러한 컨테이너는 반복자를 사용한 순차적 접근을 허용하면서 요소에 임의적 접근하는 것을 차단할 수 있다.



## 5.4 인덱스 대신 반복자 사용하기

인덱스를 사용하지 않고 다음처럼 extract_fails 함수를 구현할 수 있다.

```c++
vector <Student_info> extract_fails(vector<Student_info>& students)
{
	vector<Student_info> fails;
	vector<Student_info>::iterator iter = students.begin();
	while (iter != students.end()) {
		if (fgrade(*iter) {
			fails.push_back(*iter);
			iter = students.erase(iter);
		}
		else
			++iter;
	}
	return fails;
}
```

우선 이전 버전과 같이 fail 벡터를 정의하고 students벡터 요소를 탐색하기 위해 반복자 iter를 정의한다.

```c++
vector<Student_info>::iterator iter = students.begin();
```

iter는 erase 함수를 호출하여 students 벡터를 수정할 때 사용할 것이기에 non_const타입을 사용하며, while문은 students 벡터의 모든 요소를 탐색할 때까지 실행된다. 

iter는 컨테이너를 가리키는 반복자 이므로 *iter가 해당 요소 값이다.



students.erase(iter)를 호출하면 iter가 가리키는 요소가 사라졌으므로 iter는 더 이상 같은 요소를 참조할 수 없다.

사실 erase 함수를 호출하면 벡터에서 제거된 요소의 순서상 다음 요소을 참조하는 모든 반복자를 무효로 만든다. erase하면 뒤에 있는 값들이 앞으로 당겨지기 때문이다. 그러므로 it는 이제 엉뚱한 위치를 가리킨다.

다행이 erase 함수는 제거한 요소의 순서당 다음 요소를 가리키는 반복자를 반환한다.

```c++
iter = students.erase(iter)
```

이 코드를 실행하면 iter는 제거한 요소의 순서상 다음 요소를 가리키게 된다.

해당 요소가 과목 기준 점수를 통과하였다면 iter를 증가시켜 순서상 다음 요소를 카리키도록 한다. 이 부분은 else에서 이루어 진다.



이런 실수를 주의하자.

while문을 순환할 때마다 students.end()값을 구해야 한다. 미리 students.end()값을 구하지 말자.

```c++
vector <Student_info>::iterator iter = students.begin(), end_iter = students.end();
while (iter != end_iter){
 // 중략
}
```

미리 이야기 했든 erase함수를 실행하면 제거한 요소의 순서상 다음 요소들을 참조하는 모든 반복자를 무효로 만들며 여기에 end_iter도 포함기 때문이다.



## 5.4 더 나은 성능을 위한 데이터 구조

지금까지 구현한 프로그램은 입력 데이터양이 적으면 제대로 동작하지만 데이터 양이 늘어나면 성능이 크게 나빠진다.

벡터 내부에 요소를 삽입하거나 제거하는 경우 삽입 또는 제거한 곳 이후에 모든 요소가 이동해야 한다.

이러한 성능의 한계를 극복하려면 컨테이너의 어느 곳에서느 효율적으로 요소를 삽입 또는 제거할 수 있는 데이터 구조가 필요하다.



## 5.5 리스트 타입

컨테이너 내부에서 효율적으료 요소를 제거하는 데이터구조를 사용하도록 프로그램을 다시 만들어 보자

라이브러리에서는 이러한 종류의 접근에 최적화된 **리스트**라는 타입을 제공한다. <list>에 정의 되어 있다.

벡터가 빠른 임의적 접근에 최적화 된 것처럼 리스트는 내부 어디든 빠르게 삽입하고 제거하는데 최적화 되어 있다.

대신, 더 복잡한 구조이므로 순차적 접근만 지원할 때는 리스트가 벡터보다 느리다.

리스트는 벡터와 마찬가지로 거의 모든 타입의 객체를 담을 수 있다. 

리스트는 인덱스를 지원하지 않는다.

```c++
list <Student_info> extract_fails(list<Student_info>& students)
{
	list<Student_info> fails;
	list<Student_info>::iterator iter = students.begin();
	while (iter != students.end()) {
		if (fgrade(*iter) {
			fails.push_back(*iter);
			iter = students.erase(iter);
		}
		else
			++iter;
	}
	return fails;
}
```

위의 코드에 차이점은 함수를 호출할 때 list를 전달해야 하고 리스트를 반환하는 차이점이 있다.



### 5.1.1 리스트 vs 벡터

벡터는 erase를 사용할 경우 제거한 요소와 순서상 이어지는 요소들을 참조하는 모든 반복자를 무효로 만든다.

push_back함수를 사용하여 추가해도 전부 무효가 된다. 요소를 제거하면 순서상 이어지는 모든 요소가 이동하고, 요소를 추가하면 해당 요소를 위한 새 공간을 마련하고 벡터 전체를 재할당 하기 때문이다.



리스트 클래스의 반복자는 임의로 접근할 수 없다. 지금 당장은 표준 라이브러리가 제공하는 sort함수를 사용할 수 없다는 것만 기억하자.

그 대신 리스트 클래스는 자체적으로 멤버함수 sort를 제공한다.

```c++
vector<Student_info> students;
sort(students.begin(), students.end(), compare);

list<students_info> students;
students.sort(compare);
```

### 5.5.2 리스트를 사용해야 하는 이유

- 컨테이너 요소에 순차적으로 접근하는 경우 -> 벡터
- 컨테이너 내부 요소를 제거하는 경우 -> 리스트



## 5.6 문자열 분할

이제는 문자열을 구체적으로 알아보자.

문자열에서 특정 문자를 찾아야할 때를 생각하자.

문자열은 특별한 종류의 컨테이너로도 생각할 수 있다. 문자열은 문자로만 구성되어 있고, 전부는 아니지만 컨테이너 연산 몇 가지를 지원한다.

이러한 연산에서 인덱스를 사용할 수 있고, 문자열 타입은 벡터처럼 반복자를 제공한다. 따라서 벡터에 적용할 수 있는 다양한 연산을 문자열에도 적용할 수 있다.

예를 들어 문자열을 하나의 공백으로 구분하여 각각의 단어로 분리할 수 있다. 그리고 사용자가 입력한 문장에 직접 접근하여 해당 문자열에서 각각의 단어를 손쉽게 얻을 수 있다.

함수 문자열을 전달받아 공백으로 구분하여 분리한 단어를 요소로 갖는 vector<string>을 반환한다고 생각해 보자.

함수에서 문자열에서 분리할 단어의 범위를 정하려고 인덱스 i,와 j를 정의할 것이다.

```c++
vector<string> split(const string& s)
{
	vector<string> ret;
	typedef string::size_type string_size;
	string_size i = 0;

	while (i != s.size()) {
		// 선행하는 공백들을 부시

		//불변성 : [원래의 i, 현재의 i) 범위에 있는 문자들은 모두 공백
		while (i != s.size() && isspace(s[i]))
			++i;

		// 순서상 다음 단어의 끝을 찾음
		string::size_type j = i;

		//불변성 : [현재의 j, 다음의 j) 범위에 있는 문자들은 모두 공백이 아님

		while (j != s.size() && !isspace(s[j]))
			++j;

		//공백이 아닌 문자들을 찾았을 때
		if (i != j) {
			// i부터 j-i개의 문자를 s에 복사
			ret.push_back(s.substr(i, j - i));
			i = j;
		}
	}
	return ret;
}
```

isspace함수를 정의하는 <cctype> 헤더가 필요하다. 개별 문자를 처리하는데 유용한 함수들을 정의한다.

split함수의 매개변수 s는 단어를 추출하여 복사하므로 바꿀 필요가 없다.

i는 각 단어의 시작점을 찾는데 사용하는 인덱스이다. 가장 바깥쪽 while문을 실행할 때마다 함수에 전달한 문자열에서 한 번에 하나의 단어만큼 이동하고 문자열의 마지막 단어를 처리한 후 실행을 마친다.



while문 안에서는 2개의 인덱스를 다룬다. 처음에는 문자열 s에서 현재 i가 가리키는 위치 혹은 그 이후의 위치에 있는 공백이 아닌 첫 번째 문자를 찾는다. 여러개의 공백 문자가 연속해서 있을 수 있으므로 공백이 아닌 문자가 나타날 때까지 i를 증가시킨다.

```c++
while (i != s.size() && isspace(s[i]))
    ++i;
```

 isspace 함수는 char타입 객체를 전달 받아 공백인지 아닌지 나타내는 값을 반환한다.

이 표현은 i가 문자열 끝에 도달하지  않고 s[i]가 공백문자일 때까지 참을 반환한다.

i가 유효한 인덱스라면 그 다음 while문에서는 문자열 s에서 현재 단어의 끝을 나타내는 공백을 찾는다. 먼저 인덱스 j를 생성하고 i 값으로 초기화 한다.

```c++
while (j != s.size() && !isspace(s[j]))
	++j;
```

앞서 살펴몬 while문과 비슷하게 실행하지만 공백 문자를 만나면 while문에서 빠져 나온다. 우선 인덱스 j가 여전히 문자열 s의 범위에서 벗어나지 않았는지 확인하고 j가 유효하면 isspace함수를 호출한다. 이번에는 **논리 부정 연산자**를 사용하여 isspace함수의 반환 값을 부정한다. 즉, isspace(s[j])가 참이 아니면 참으로 만든다.



내부에서 2개의 while문을 거치면 하나의 단어를 찾았거나 단어를 찾는 동안 문자열 마지막까지 검사했을 것이다.

만약 문자열 검사가 끝났으면 i와 j 모두 s.size()와 같아진다. 그게 아닐 경우 ret 벡터에 추가해야하는 단어를 찾은 것이다.



```c++
//공백이 아닌 문자들을 찾았을 때
if (i != j) {
    // i부터 j-i개의 문자를 s에 복사
    ret.push_back(s.substr(i, j - i));
    i = j;
}
```

substr 함수는 문자열 클래스의 내부 함수로 인덱스와 길이를 인수로 전달한다.

첫 번째 인수에서 지정한 인덱스로 부터 두번째 인수가 나타내는 수만큼 문자를 복사하여 새로운 문자열을 만든다.

문자열 s에서 인덱스가 i인 문자부터 시작하여 반 개방 범위인 [i,j)안 모든 문자를 복사한다.



## 5.7 split 함수 테스트

split 함수를 테스트 하는 프로그램이다.

```c++
int main()
{
	string s;
	while (getline(cin, s)) {
		vector<string> v = split(s);
		for (vector<string>::size_type i = 0; i != v.size(); ++i)
			cout << v[i] << endl;
	}
}
```

이 프로그램은 한 번에 한 행씩 읽어야 한다. 이를 위해 문자열 라이브러리는 행 끝에 도달할 때까지 입력을 읽을 수 있도록 getline 함수를 제공한다.

getline은 2개의 인수가 있는데 첫번째는 입력데이터에 대한 istream타입이고, 두 번째는 입력 데이터를 저장할 문자열의 참조 타입이다.

보통 getline함수는 istream의 참조타입을 반환하므로 while문 조건에서 istream상태를 확인할 수 있다.

만약 EOF에 도달하거나 유효하지 않는 입력이 발생하면 false를 반환하고 while문에서 벗어난다.



## 5.8 문자열 결합

이전에 본 누군가의 이름을 별표로 구성한 테두리로 묶어 출력하는 프로그램을 생각해 보자.

이 문제를 테두리로 묶인 문자열 전체를 저장하는 단일 데이터 구조를 만드는데 초점을 맞춰서 살펴 보자.



### 5.8.1 테두리 입히기

vector<string>에 저장한 모든 단어를 행 하나하나 출력하고 모든 문자열을 테두리로 둘러 싸보자.

별표와 단어 사이에 공백 하나를 남긴 채 왼쪽 테두리에 맞춰 정렬해보자

p를 'this is an', 'example', 'to', "illustrate", 'framing'을 포함하는 벡터라고 하고 frame(p)를 호출하면 vector<string> 타입을 반환하는 frame함수를 만들어 보자 frame함수가 반환한 벡터요소를 차례로 출력한 결과는 다음과 같다.

```c++
**************
* this       *
* is         *
* an         *
* example    *
* to         *
* illustrate *
* framing    *
**************
```





각 문자열의 길이는 서로 다르지만 테두리는 직사각형 모양이다. 즉 벡터에서 가장 긴 문자열을 찾는 함수가 필요하다.

```c++
string::size_type width(const vector<string>& v)
{
	string::size_type maxlen = 0;
	for (vector<string>::size_type i = 0; i != v.size(); ++i)
		maxlen = max(maxlen, v[i].size());
	return maxlen;
}
```

벡터를 걸쳐 가장 긴 문자열의 요소를 찾아 maxlen에 저장한다.



frame 함수를 만들어 보자.

```c++
vector <string> frame(const vector<string>& v)
{
	vector<string> ret;
	string::size_type maxlen = width(v);
	string border(maxlen + 4, '*');

	// 상단 테두리를 추가
	ret.push_back(border);
	
	// 양 끝이 별표와 공백으로 둘러싸인 문자열들을 각각 추가
	for (vector<string>::size_type i = 0; i != v.size(); ++i)
		ret.push_back("* " + v[i] + string(maxlen - v[i].size(), ' ') + " *");
	
	// 하단 테두리를 추가
	ret.push_back(border);
	return ret;
}
```

전달할 벡터를 수정하지 않으므로 const 참조 타입, vector<string>을 반환하며 이를 위해 함수 안에 ret을 만든다.

width를 사용해서 가장 긴 문자열을 찾은 후 , 여러개의 별표로 구성된 문자열을 만든다.

```c++
int main()
{
	string s;
	while (getline(cin, s)) {
		vector<string> v = split(s);
		vector<string> f = frame(v);

		for (vector<string>::size_type i = 0; i != f.size(); ++i)
			cout << f[i] << endl;
	}
}
```



### 5.8.2 수직 결합

지금까지는 문자 그림을 1개만 출력하는 방법을 살표보았는데, 지금부터는 수평으로 여러 개 문자 그림을 결합하는 것을 살펴보자.

문자 그림을 저장하는 vector<string>은 요소 각각이 행 하나와 같다. 따라서 문자 그림 2개를 수직으로 결합하는 것은 간단하다. 단순히 해당 2개를 결합하면 된다.

```c++
vector<string> vcat(const vector<string>& top, const vector<string>& bottom)
{
	// 위쪽 문자 그림을 복사
	vector<string> ret = top;

	//아래쪽 문자 그림을 복사
	for(vector<string>::const_iterator it = bottom.begin(); it != bottom.end(); ++it)
		ret.push_back(*it);
	return ret;
}
```

for문에서는 하나의 컨테이너에서 다른 컨테이너로 요소의 복사본을 삽입하는 일반적인 개념을 구현했다.

라이브러리는 반복문을 사용하지 않고 연산하도록 만들 수 있다.

```c++
for(vector<string>::const_iterator it = bottom.begin(); it != bottom.end(); ++it)
    ret.push_back(*it);

// 이걸로 변경 가능
ret.insert(ret.end(), bottom.begin(), bottom.end());
```



### 5.8.3 수평 결합

두개의 문자열 그림을 좌우로 붙여 보자. 두 그림의 각 행을 나란히 이어 붙일 수 있다.

```
// 예시
A           B
C     +     D
```

```
// 결과
A B
C D
```

이런식으로 결합할 것이며 두 그림의 행 개수(문자 수)가 다르면 (왼쪽 : 5줄, 오른쪽 : 3줄)

부족한 쪽은 공백으로 채워야 한다.

만약 문자열 p와 frame(p)를 결합하면

```
this is an ***************
example    * this is an *
to         * to         *
illustrate * illustrate *
framing    * framing    *
           ***************
```

이렇게 나와야 한다.

```c++
vector <string> hcat(const vector<string>& left, const vector<string>& right)
{
	vector<string> ret;

	// 두 문자 그림 사이에 공백을 하나 남김
	string::size_type width1 = width(left) + 1;
	
	// 왼쪽 문자 그림과 오른쪽 문자 그림의 요소를 살펴보는 인덱스
	vector<string>::size_type i = 0, j = 0;

	// 두 문자 그림의 모든 행을 살펴봄
	while (i != left.size() || j != right.size()) {

		// 두 문자 그림의 문자들을 처리할 새로운 문자를 생성
		string s;

		// 왼쪽 문자 그림에서 행 하나를 복사
		if (i != left.size())
			s = left[i++];

		// 공백을 포함하여 문자열을 적절한 너비까지 채움
		s = string(width1, ' ');


		// 오른쪽 문자 그림에서 행하나를 복사
		if (j != right.size())
			s += right[j++];

		//새로운 문자 그림에 문자열 s를 추가
		ret.push_back(s);
	}
	return ret;
}
```



## 5.9 핵심정리

**컨테이너와 반복자**

표준 라이브러리는 서로 다른 컨테이너에서 동작하는 비슷한 연산들이 같은 인터페이스를 사용하고 의미도 같게 설계되었다.

지금까지는 **순차 컨테이너**를 다루었다. 모든 순차 컨테이너와 문자열 타입은 다음과 같은 연산을 제공한다.

- 컨테이너<T>::iterator 또는 컨테이너<T>::iterator : 현재 컨테이너가 지원하는 반복자 타입의 이름이다.
- 컨테이너<T>::size_type : 현재 컨테이너의 가장 큰 인스턴스 크기를 저장하기에 알맞은 타입의 이름이다.
- c.begin() 또는 c.end()  : 컨테이너의 첫 번째 요소를 참조하는 반복자와 마지막 요소의 다음을 참조하는 반복자입니다.
- c.rbegin() 또는 c.rend() : 컨테이너 요소에 역순으로 접근하려고 컨테이너의 마지막 요소를 참조하는 반복자와 첫 번째 요소 이전을 참조하는 반복자.
- 컨테이너<T> c; 또는 컨테이너<T> c(c2);  c를 비어 있는 컨테이너로 정의하거나 주어진 c2의 복사본으로 정의한다.
- 컨테이너<T> c(n); : c를 T의 타입에 따라 값 초기화된 n개 요소가 있는 컨테이너로 정의한다. 만약 T가 클래스 타입이면 해당 타입이 요소의 초기화 방식을 제어합니다. 만약 T가 기본 산술 타입이면 요소는 0으로 초기화됩니다.
- 컨테이너<T> c(n, t); : c를 t의 복사본인 n개 요소가 있는 컨테이너로 정의한다.
- 컨테이너<T> c(b, e); : [b, e) 범위의 반복자가 가리키는 요소들의 복사본이 있는 컨테이너를 만듭니다.
- c = c2; : 컨테이너 c의 내용을 컨테이너 c2의 복사본으로 대체한다.
- c.size();  : c에 있는 요소 개수를 size_type 타입으로 반환한다.
- c.empty(); : c에 요소가 없는지 나타내는 서술 함수.
- c.insert(d, b, e); : [b, e) 범위의 반복자가 가리키는 요소들을 복사하고 컨테이너 c에서 d가 가리키는 위치에 삽입
- c.erase(it) 또는 c.erase(b, e);  : 컨테이너 c에서 it이 가리키는 요소를 제거하거나 [b, e) 범위가 가리키는 요소들을 제거. 제거한 요소 바로 다음 위치를 참조하는 반복자를 반환한다.. 이 연산은 리스트를 사용하면 빠르지만 벡터와 문자열에서 사용하면 느릴 수 있다. 왜냐하면 벡터와 문자열은 요소가 제거된 후 모든 요소를 복사하는 과정을 포함하기 때문.

- c.push_back(t); : t 값이 있는 요소를 컨테이너 c의 끝에 추가합니다



**반복자 연산**

- *it : 반복자 it를 역참조 하여 가리키는 위치에 저장된 컨테이너 요소 값을 얻는다.
- it->x : (*it).x와 같은 의미이다. 반복자 it를 역참조하여 얻은 객체의 멤버인 x를 반환한다. 
- ++it 또는 it++ : 반복자를 증가시켜 컨테이너의 순차상 다음 요소를 가리키도록 한다.
- b == e 또는 b != e  : 두 반복자가 같은지를 비교합니다.



**문자열 타입**

벡터의 반복자와 같은 연산을 실행하는 반복자를 제공한다. 문자열은 컨텡너에서 제공하는 연산 외에도 다음과 같은 연산을 제공한다.

- s.substr(i, j); : [i, i + j) 범위의 인덱스를 사용하여 문자열 s에 있는 문자들의 복사본을 저장하는 새로운 문자열을 생성.
- getline(is, s); : is에서 행 하나의 입력을 읽어 s에 저장.
- s += s2; : s 값을 s + s2의 값으로 대체



**벡터 타입**

모든 라이브러리 컨테이너 중 가장 강력한 반복자인 임의 접근 반복자를 제공한다. 

- v.reserve(n); : n개 요소를 저장할 저장 공간을 예약하되 초기화하지는 않는다. 이 연산은 컨테이너의 크기를 바꾸는 것이 아니며. 벡터가 메모리를 할당하려고 insert 함수나 push_back 함수를 반복적으로 호출하는 횟수에만 영향을 줍니다.
- v.resize(n); : v의 크기를 n으로 새롭게 지정합니다. 만약 n이 v의 현재 크기보다 작다면 n 이후의 요소는 벡터에서 제거된다. 만약 n이 v의 현재 크기보다 크다면 새로운 요소들이 v에 추가되고 v의 타입에 따라 알맞게 초기화된다.



**리스트 타입**

컨테이너의 어느 지점에서나 효율적으로 요소를 삽입하고 삭제할 수 있도록 최적화되어 있다. 

- l.sort() 또는 l.sort(cmp);  : 리스트에 있는 타입의 < 연산자나 서술 함수 cmp를 사용하여 l의 요소들을 정렬한다.



**<cctype> 헤더**

문자 데이터 조작에 유용한 함수들을 제공한다.

- isspace(c); : c가 공백 문자일 때 참
- isalpha(c); : c가 알파벳 문자일 때 참
- isdigit(c); : c가 숫자 문자일 때 참
- isalnum(c); : c가 문자나 숫자일 때 참
- ispunct(c); : c가 구두 문자일 때 참
- isupper(c); : c가 대문자일 때 참
- islower(c); : c가 소문자일 때 참
- toupper(c); : c에 해당하는 대문자 반환
- tolower(c); : c에 해당하는 소문자 반환
