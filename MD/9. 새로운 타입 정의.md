# 9. 새로운 타입 정의

C++의 타입은 기본 타입과 클래스 타입으로 나눌 수 있다.

기본 타입은 프로그래밍 언어의 기본 영역에 정의되어 있으며 char, int, double 등이 있다.

반면 라이브러리를 이용한 타입인 문자열, 벡터, istream등은 모두 클래스 타입이다.

C++에서 설계한 기능 대부분은 기본 타입을 이용하여 손쉽게 프로그래머가 새로운 타입을 만들 수 있게 하자는 개념을 기초로 한다.

9장에서는 기본적인 클래스의 정의 기능을 살펴보자



## 9.1 Student_info 살펴보기

4.2.1에서는 학생들의 과목 점수를 다루는 프로그램을 쉽게 작성하려고 Student_info라는 데이터 구조와 함수를 만들었다.

하지만 이렇게 만든 구조는 다른 프로그래머사 사용하기에 적합하지 않다.

우리가 작성한 함수를 사용하려는 프로그램은 특정 규칙을 따라야 한다. 예를 들어 새로 만든 Student_info 객체를 사용하려면 먼저 데이터를 읽어 들어야 한다. 그렇지 않으면 빈 homework 벡터와 정의하지 않는 값이 할당된 midterm과 final이 있는 객체가 만들어지는데, 이런 객체를 사용하면 부정확한 결과를 얻거나 갑작스러운 충돌이 일어날 수 있다.

또한 기존 Student_info에 관한 '인터페이스'가 흩어져 있다. 기존 방법처럼 read 함수 같은, Student_info 객체의 상태를 바꾸는 함수를 하나의 헤더파일에 넣을 수 있다. 하지만 이러한 방법은 선택사항이며 그룹화를 위한 요구사항은 존재하지 않는다.



## 9.2 클래스 타입

클래스 타입은 연관된 데이터 값을 데이터 구조에 결합한 상태이다. 해당 데이터 구조는 독립적이다. 예를 들어 Student_info 구조체는 Student_info 타입 객체를 정의하고 조작할 수 있다.

```c++
struct Student_info {
	std::string name;
	double midterm, final;
	std::vector<double> homework;
};
```

Student_info 타입의 각 객체에는 4가지 요소가 있다. Student_info 타입을 사용하는 프로그래머는 이러한 데이터 요소를 직접 조작해야 한다.

Student_info 정의에서 데이터 요소에 접근을 제한하지 않으므로 프로그래머는 데이터를 직접 조작할 수 있다.

Student_info 타입을 사용하는 프로그래머가 직접 데이터에 접근하는 것을 허용하지 않도록 구현 사항을 숨길 수 있다. 특히 함수로만 객체에 접근하도록 만들 수 있다.

그러려면 먼저 Student_info 객체를 조작하기에 적합한 함수를 사용자에게 제공해야 한다. 이 함수는 클래스의 인터페이스를 구성하게 된다.



### 9.2.1 멤버 함수

Student_info 객체의 접근을 제어하려면 프로그래머가 사용할 수 있는 인터페이스를 정의해야 한다. 우선 정보를 읽는 동작과 최종 점수를 구하는 동작을 정의해보자.

```c++
struct Student_info {
	std::string name;
	double midterm, final;
	std::vector<double> homework;

	std::istream& read(std::istream&);
	double grade() const;
};
```

2개의 멤버함수를 추가하였다.



제네릭 함수를 지원하는 프로그래밍 언어는 이해하기 쉽다. 단 find 함수가 '적절한 타입의 인수를 허용'한다는 의미를 이해해야 하는데 예를 들어 find 함수가 어떤 방식으로 동작하는지 어떻게 설명해야 할까?

이에 대한 답은 C++ 언어 내부에 있고 일부는 언어 밖에 있다.



C++ 언어 내부에서의 답은 다음과 같다.

함수가 **알 수 없는 타입의 매개변수를 어떻게 사용하는지**가 그 매개변수의 타입을 제한한다.

예를 들어, 어떤 함수가 x와 y를 매개변수로 받아서 x + y를 계산한다면,

이 연산이 존재한다는 사실 자체가 x와 y는 반드시 `+` 연산이 가능한 타입이어야 함을 의미한다.

이러한 함수를 호출할 때마다, 구현체는 **전달한 인자의 타입이 함수에서 요구하는 조건을 만족하는지 확인한다.**



C++ 언어 밖에서의 답은 표준 라이브러리가 함수의 매개변수에 대한 조건을 어떻게 정리하는지에 있다.

이미 한 가지 예를 보았는데, 그것이 바로 **반복자**라는 개념이다.어떤 타입은 반복자이고, 어떤 타입은 그렇지 않다.
 `find` 함수는 세 개의 인자를 받는데,
 👉그중 앞의 두 개는 반드시 이터레이터여야 한다.

즉 특정 타입을 반복자로 지정하는 것은 해당 타입이 지원하는 연산 내용을 지정하는 것이다.

만약 fine 함수를 직접 작성할 때는 반드시 모든 반복자를 지원하는 연산에만 지원하는  방식일 것이다.



### 8.1.1 알려지지 않은 타입의 중앙값

제네릭 함수를 구현하는 프로그램 언어의 특징을 **템플릿 함수**를 사용한다고 말한다.

템플릿을 사용하면 **템플릿 매개변수**의 타입은 다르지만 동일하게 동작하는 함수(또는 타입) 집합의 정의를 단일화할 수 있다.

템플릿의 핵심 개념은 서로 다른 타입의 객체가 공통 동작을 공유할 수 있다는 것인데, 템플릿 매개변수를 사용하면 템플릿을 정의할 때 템플릿 매개변수에 해당하는 명확한 타입을 알지 못하더라도 공통 동작을 실행하는 프로그램을 작성할 수 있다.

템플릿을 사용할 때 비로소 타입을 알 수 있으며 프로그램의 컴파일 및 링크단계에서 정보를 얻을 수 있다.



4.1.1에 작성한 함수를 예로 들어보면 이 함수는 벡터의 요소의 중앙값을 구하는 함수를 예를 들어보면 이 함수는 벡터를 정렬한 다음 주어진 인덱스에 특정 요소를 추출하는 동작을 실행한다.

다음 처럼 템플릿 함수를 사용하면 서로 다른 타입의 요소가 있는 벡터의 중앙값도 얻을 수 있다.

```c++
template <class T>
T median(vector<T> v)
{
    typedef typename vector<T>::size_type vec_sz;
    vec_sz size = v.size();
    if (size == 0)
        throw std::domain_error("median of an empty vector");
    std::sort(v.begin(), v.end());
    vec_sz mid = size / 2;
    return size % 2 == 0 ? (v[mid] + v[mid - 1]) / 2 : v[mid];
}
```

우선 살펴볼 개념은 다음과 같은 템플릿 헤더와 median 함수의 매개변수 목록 및 반환 타입에서 T를 사용하는 것이다.

```c++
template <class T>
```

템플릿 헤더는 우리가 템플릿 함수를 정의하는 것과 해당 함수에 **타입 매개변수**가 있다는 것을 구현체에게 알려주는 역할이다.

타입 매개변수는 함수 범위 안에서 사용할 수 있는 이름을 정의하고 벼눗가 아닌 타입을 참조한다. 따라서 함수 안에서 T가 나올 때마다 구현체는 T를 타입으로 인식한다.

median 함수는 타입 매개변수를 사용하여 v 벡터에 포함된 객체의 타입과 median 함수의 반환 타입을 명시한다.

median 함수를 호출하면 구현체는 T에 타입을 바인딩 시킨다. 해당 타입은 컴파일 과정에서 median 함수를 호출하는 시점에 결정된다. 

예를 들어 vector<int> 타입의 객체인 vi벡터의 중앙값을 구하려고 median(vi)를 호출할 때를 구현체는 T가 int라고 추정할 수 있다.

median 함수에서 T를 사용할 때마다 구현체는 int로 작성한 것처럼 코드를 만든다. 결과적으로 구현체는 vector<int> 타입의 객체를 취하며 int 타입 값을 반환하는 median 함수의 특정 버전을 작성한 것처럼 코드를 **인스턴스화**한다.



다음으로 살펴볼 개념은 vec_sz에서 typename을 사용하는 것이다. typename은 구현체가 T의 타입을 아직 알지 못하는 상태에서 vector<T>::size_type이 타입 이름이라는 것을 구현체에 알리는데 사용한다.

템플릿 매개변수에 따라 달라지는 vector<T>와 같은 타입이 있을 때 그 자체가 타입인 size_type같은 vector<T>의 멤버를 사용한다고 가정해보자.

전체 타입 이름 앞에 반드시 typename을 표기하여 구현체가 해당 이름을 타입으로 취급하도록 알려줘야 한다.

표준 라이브러리에는 모든 T에 vector<T>::size_type이 타입 이름임을 보장하지만 구현체는 이러한 사실을 알 방법이 없다.



반면에 vector<T> 타입인 v를 다루는 모든 연산에서는 암묵적으로 해당 타입을 사용한다. 즉 v를 쓰면 T를 굳이 안 써도 컴파일러가 알아서 사용한다.

```c++
(v[mid] + v[mid - 1])
```

v[mid]와 v[mid - 1]의 타입을 알려면 v벡터의 요소가 무슨 타입인지  알아야 한다. 그리고 해당 타입을 + 연산자와 / 연산자의 유형을 결정한다,

vector<int>타입으로 함수를 호출하면 +와 / 연산자는 int 타입의 피연산자가 있으며 int결과를 반환한다.

vector<double> 타입으로 함수를 호출하면 double 타입 값에 따른 산술연산을 실행한다.

반면 vector<string> 타입으로 실행할 수는 없다. / 연산자는 문자열 타입에 적용할 수 없기 때문



### 8.1.2 템플릿 인스턴스화  

vector<int>타입으로 함수를 호출하면 구현체는 모든 T를 int로 대체하여 템플릿 함수의 인스턴스를 만들고 컴파일 한다.

vector<double> 타입인 벡터로 호출하면 컴파일러는 호출 형태에서 타입을 유추하여 같은 동작을 실행한다.

이 때 T는 double로 바인딩 되며 구현체는 T를 double로 대체하여 median 함수의 또다른 버전을 만든다.



일반적으로 C++ 컴파일 방식(편집 -> 컴파일 -> 링크)에서는 템플릿이 컴파일 할 때가 아니라 링크할 때 만들어 지는 경우가 있다. 그래서 링크단계에서 에러가 날 수 있다.

그리고 템플릿을 직접 만들 때는 선언만 있으면 안되고 정의에도 접근할 수 있어야 한다. 

이유는 컴파일러가 템플릿을 만들려면 함수 내부 코드까지 다 알아야 하기 때문이다.

보통은 템플릿 코드를 헤더 파일에 다 넣는다.

입력을 모두 읽었다면 해당 단어의 빈도를 출력하는데 map에서 list와 vector에 차이점은 for문에서 데이터를 출력하는 방법이다.



### 8.1.3 제네릭 함수 타입

템플릿을 설계할 때 어려운 점은 템플릿이 어떤 타입에서도 동작하도록 만들기 위해 그 타입이 어떤 연산을 지원하는지 이해하는 것이다.

예를 들어 median 함수는 내부에서 이런 연산을 한다면

```c++
(v[mid] + v[mid - 1]) / 2
```

벡터 타입 T는 반드시 +와 / 연산을 지원해야 한다.



더 복잡한 문제는 템플릿과 타입 변환이 함께 작용할 때 발생한다.. 학생들이 모든 과제를 했는지 확인하려고 find를 호출했을 때를 보면

```c++
find(s.homework.begin(), s.homework.end(), 0);
```

이 경우는 homework는 vector<double>인데 우리는 int 값인 0일 찾고 있었다.

하지만 이건 문제가 안되는데 int는 double과 자연스럽게 변환될 수 있고 의미도 유지되기 때문이다.



하지만 accumulate를 사용할 때는 상황이 달라지는데

```c++
accumulate(v.begin(), v.end(), 0.0);
```

accumulate의 세번째 인수의 타입을 누적된 결과의 타입으로 사용한다. 만약 세번째 인수가 int 타입이면 double 타입 값을 더하더라도 정수 부분만 잘라서 연산을 시행하기 때문에 주의해야 한다.



마지막으로 max를 사용할 때는 

```c++
string::size_type maxlen = 0;
maxlen = max(maxlen, name.size());
```

중요한 건 maxlen의 타입과 name.size()가 반환하는 타입이 정확히 일치해야 한다. 일치하지 않으면 컴파일 되지 않는다.

이유는 max 함수는 다음과 같이 정의되어 있다.

```c++
template<class T>
T max(const T& left, const T& right);
```

즉 두 인자는 같은 T타입이어야 한다. 만약 int와 double을 함께 넘기면 어떤 타입으로 맞춰야 할지 결정할수 없기 때문에 에러가 발생한다.



## 8.2 데이터 구조 독립성

우리가 만든 median 함수에는 다양한 타입에 대해 동작하도록 만들었다. 그래서 숫자타입이라면 어떤 벡터이든 중앙값을 구할 수 있다.

더 나아가서 list, vector, string 같은 어떤 자료구조든 사용할 수 있게 만들어 보고 싶고, 전체 컨테이너가 아니라 일부 구간만 처리할 수 있도록 만들어 보려 한다면 **반복자**를 사용하면 된다.



예를 들어 표준라이브러리는 반복자를 사용하여 컨테이너의 종류와 상관없이 연속성이 있는 컨테이너 일부분에 find함수를 호출할 수 있다. 

```c++
find(c.begin(), c.end(), val);
```

왜 여기서 c를 두번 언급해야할까? 다음과 같이 쓸 수 있지 않을까?

```c++
c.find(val) 또는 find(c, val)
```

이유는 이터레이터를 쓰면 모든 컨테이너에 대해 하나의 함수로 처리 가능하게 되기 때문이다.

만약 c.fine(val) 방식이면 누군가가 반드시 멤버 함수 find를 정의해야 한다. 그리고 배열을 .find()가 없어서 사용 불가능하다.

그래서 find(begin, end, val) 구조로 통일하여 모든 컨테이너 + 배열까지 다 사용 가능하도록 한다.



또 중요한 점은 

```c++
find(c.begin(), c.end(), val)
```

```c++
find(c.begin() + 2, c.begin() + 5, val)
```

이런 구조로 작성해서 일부 구간만 검색할 수 있다.



마지막으로 반복자는 일반 컨테이너가 아닌 것도 다룰 수 있다.

예를 들어 

```c++
find(v.rbegin(), v.rend(), val)
```

이런식으로 rbegin은 컨테이너 욧에 역순으로 접근을 허용하는 반복자를 반환한다. 이러한 반복자를 find나 find_if의 인수로 전달하면 역순으로 탐색할 수 있다.

만약 find 함수나 find_if함수에 컨테이너를 직접 인수로 전달하도록 제한하면 불가능 한 동작이다.



### 8.2.1 알고리즘과 반복자

템플릿을 사용하면 자료구조에 상관 없이 동작하는 함수(알고리즘)을 만들 수 있다. 이를 이해하려면 표준 라이브러리 함수가 어떻게 구현되었는지 알면 쉽게 할 수 있다.

이러한 표준라이브러리 함수들은 인수에 반복자가 포함되어 함수가 다루는 컨테이너 요소들을 식별할 수 있다.

모든 표준 라이브러리 컨테이너와 문자열 같은 몇몇 타입은 표준 라이브러리 함수들이 컨테이너 요소들을 다룰 수 있는 반복자를 제공한다.



어떤 컨테이너는 특정 기능을 지원하고 어떤 기능은 안된다. 즉 컨테이너 마다 가능한 연산이 다르다.

예를 들어 vector는 인덱스로 접근 가능하지만 list는 불가능하다.

```c++
vector → v[3] //가능
list → l[3] //불가능
    
vector iterator → // + 가능
list iterator → // + 불가능
```

그래서 iteraotr마다 가능한 연산도 다르다.

서로 다른 iterator라도 같은 연산을 지원하면 같은 의미를 가진다. 예를 들어 ++ 연산은 항상 다음 요소를 가리킨다.

즉 자료구조가 달라도 공통 동작은 동일하게 유지된다.



모든 알고리즘이 반복자의 모든 기능을 필요로 하지 않는다. 예를 들어 

```c++
find(begin, end, val)
```

이 함수는 값을 하나씩 확인하면서 한 칸 씩만 이동하면 된다. 그래서 아주 기본적인 반복자 기능만 있으면 된다.

반면 sort 함수는 여러 위치를 이동하고 요소를 빠르게 바꾸기 때문에 일부 컨테이너에서만 사용 가능하다.



라이브러리는 5개의 **반복자 카테고리**를 정의하며 각 카테고리는 특정 반복자 연산들을 모아놓은 것이다.

반복자 카테고리는 라이브러리 컨테이너마다 지원하는 반복자를 종류별로 분류하는데, 각 표준 라이브러리 알고리즘은 정해진 카테고리에 속한 반복자를 인수로 전달 받는다.

따라서 반복자 카테고리를 이용해 어떤 컨테이너가 어떤 알고리즘을 사용하는지 이해할 수 있다.



각 알고리즘은 자신이 요구하는 iterator의 종류를 결정한다. 이 카테고리를 통해 어떤 컨테이너에서 어떤 알고리즘을 사용할 수 있는지 결정된다.

예를 들어 일부 알고리즘은 입력 데이터에 단일 방향의 접근만 지원한다면 여러 방향의 접근을 지원하는 반복자가 필요하지 않다.



지금 부터 접근 방식 각각과 이에 사용하는 알고리즘을 소개한 다음 해당 반복자 카테고리를 설명할 것이다.



### 8.2.2 순차적 읽기 접근

순차 컨테이너에 접근하는 간단한 방법은 요소를 순차적으로 읽는 것이다. 그러한 동작을 실행하는 라이브러리 함수엔find가 있고 다음처럼 구현할 수 있다.

```c++
template <class In, class x>
In findx(In begin, In end, const x& value)
{
    while (begin != end && *begin != value)
        ++begin;
    return begin;
}
```

만약 find(begin, end ,x) 형태로 함수를 호출하면 [begin, end) 범위에서 *iter == x를 만족하는 첫 번째 반복자인 iter를 반환한다. 그러한 반복자가 존재하지 않으면 end를 반환한다.

begin 값을 바꾸는데 사용하는 유일한 연산이 ++이므로 find 함수가 begin,end 범위의 요소에 순차적으로 접근한다는 것을 알 수 있다.

그리고 begin과 end를 비교하려고 != 를 사용하고 begin이 참조하는 컨테이너 요소에 접근하려고 *를 사용한다. 이러한 연산들을 범위 안 반복자 값이 참조하는 요소를 순차적으로 읽기 충분하다

앞에서 언급한 모든 연산을 제공하는 반복자의 유형을 **입력 반복자**라고 한다.

입력 반복자는 데이터를 한 번씩 읽으면서 앞으로만 이동할 수 있는 반복자이며

```c++
*it      // 값 읽기
++it     // 다음으로 이동
it != end // 비교
```

이 3개만 가능하면 입력 반복자이다.

대표적인 예시로

```c++
vector<int>::iterator
list<int>::iterator
int* p
```

가 있다



### 8.2.3 순차적 쓰기 접근

입력 반복자는 순차적인 요소를 읽을 때만 사용할 수 있다. 그리고 순차 컨테이너의 요소를 쓸 때 반복자를 사용해야 하는 상황이 있을 수 있다. copy 함수를 예로 들어 보자

```c++
template <class In, class Out>
Out copy(In begin, In end, Out dest)
{
    while (begin != end)
        *dest++ = *begin++;
    return dest;
}
```

copy 함수는 3개의 반복자가 인수로 있다. 2개의 반복자는 순차컨테이너를 나타내고 세 번째 반복자는 목적지에 해당하는 순차 컨테이너의 시작 지점을 나타낸드ㅏ.

begin에서 시작하여 end에 도달할 때까지 컨테이너를 통과하여 각 요소를 dest로 복사하는 동작을 실행한다.

In이라는 타입이름에서 알 수 있듯이 begin과 end는 입력 반복자이며 읽는 용도로만 사용한다.

dest의 타입인 out이 어떤 연산을 실행하는지 살펴보면 *dest = 값rhk dest++를 발견할 수 있다. 그리고 ++dest와 같은 연산을 할 수 있어야 한다.



앞 코드에서 드러나지  않은 요구사항을 한 가지 살펴보자 it를 쓰기 전용으로 사용하려는 반복자라 가정하고 다음 코드를 살펴 보자

```c++
*it = x;
++it;
++it;
*it = y;
```

*it에 값을 할당하는 표현식 사이에 it를 두 번 증가시켜 목적지에 해당하는 순차컨테이너의 간격을 남겼다.

반복자를 쓰기 전용으로 사용하려면 *it에 값을 할당하는 표현식 사이에 ++it를 두 번 이상 실행하지 않거나 it를 증가시키지 않은 채로 두 번 이상 *it에 값을 할당하지 않는 암묵적인 요구사항이 있어야 한다.

즉 3가지 규칙이 있어야 하는데

1. 값을 쓸때마다 항상 한 번씩 이동해야 함
2. 이동 없이 2 번 쓰지 말 것
3. 쓰기 사이에 여러번 이동하지 말 것

이 암묵적인 요구사항을 충족하는 방식으로 사용할 때 해당 반복자의 유형을 **출력 반복자**라고 한다.

모든 표준 컨테이너는 back_inserter 함수의 반환 값처럼 암묵적인 요구사항을 따르는 반복자를 사용한다.

참고로 모든 변수의 한 번의 할당만을 허용하는 것은 iterator 자체의 제약이 아니라 iterator를 사용하는 프로그램의 제약이다.

즉 output iterator는 두 번 사용하면 에러가 나는게 아니라 결과가 이상해 질 수 있다.



### 8.2.4 순차적 읽기 및 쓰기 접근

이번엔 데이터를 순차적으로 읽고 쓰는 방식을 생각해보자, 중요한 건 앞으로만 이동하고 뒤로는 가지 않는 것이다.

<algorithm>에 정의된 replace 함수의 예를 들어 보자

```c++
template <class For, class X>
void replace(For beg, For end, const X& x, const X& y)
{
    while (beg != end) {
        if (*beg == x)
            *beg = y;
        ++beg;
	}
}
```

replace 함수는 [beg, end) 범위의 모든 요소를 탐색하여 x인 요소 값을 y로 바꾼다.

For 타입의 반복자는 입력 반복자와 출력 반복자의 모든 연산을 제공해야 한다.

또한 출력반복자의 요구 사항인 모든 변수의 한 번의 할당만을 허용하는 속성을 충족시킬 필요도 없다. 같은 것만 바꾸면 되기 때문

이러한 반복자의 유형을 **순방향 반복자**라고 하며 다음 연산을 지원해야 한다.

```c++
*it        // 읽기
*it = val  // 쓰기
++it       // 이동
it->member // 접근
```

모든 표준 라이브러리 컨테이너는 순방향 반복자의 요구 사항을 충족시킨다.

즉 순방향 반복자는 읽기 + 쓰기 + 앞으로 이동이 모두 가능한 반복자이다.



### 8.2.5 양방향 접근

어떤 함수는 컨테이너 요소를 역순으로 얻어야 한다. reverse 함수를 예시로 들어 보자.

```c++
template<class Bi>
void reverse(Bi beg, Bi end)
{
    while (beg != end) {
        --end;
        if (beg != end)
            swap(*beg++, *end);
    }
}
```

end 반복자가 벡터 끝에서 시작해 역방향으로 이동하고 반복자 beg가 처음에서 시작해 순방향으로 이동하며 두 반복자가 참조하는 요소를 교환한다.

-- 를 사용하는 점 제외하면 begin과 end를 순방향 반복자 처럼 사용한다. --는 순차 컨테이너를 역방향으로 탐색할 수 있도록 돕는 역할이며, 이처럼 순방향 반복자의 모든 요구사항을 충족하고 --를 지원하는 반복자 유형을 **양방향 반복자**라고 한다. 모든 표준 라이브러리 컨테이너 클래스는 양방향 반복자를 지원한다.



### 8.2.6 임의적 접근

어떤 함수들은 컨테이너 내부를 자유롭게 탐색할 수 있어야 한다.

대표적으로 이진 탐색 알고리즘이 있다. 이진 탐색이란 정렬된 데이터에서 원하는 값을 빠르게 찾는 방법이다.

```c++
가운데를 본다 → 반으로 버린다 → 반복
```

즉 중간으로 바로 가야 한다 한칸씩 ++하면 안된다.

```c++
template<class Ran, class X>
bool binary_search(Ran begin, Ran end, const X& value)
{
    while (begin < end) {
        // 범위의 중간 지점 찾기
		Ran mid = begin + (end - begin) / 2;

        // 중간 지점을 기준으로 어느 부분이 x를 포함하는지 확인하고
        // 해당 부분에만 탐색을 진행
        if (x < *mid)
            end = mid;
        else if (*mid < x)
            begin = mid + 1;
		// 이 지점에 도달하면 *mid == x 이므로 탐색  종료
        else
			return true; // *mid == x
    }
}
```

binary_search 함수는 앞서 살펴본 다른 반복자의 속성을 필요로 하며 추가적으로 반복자의 산술 연산을 필요로 한다.

예를 들면 정수값을 얻으려고 반복자를 다른 반복자로 빼기 연산을 하거나 또 다른 반복자를 얻으려고 반복자에 정수 값을 더하는 기능이 필요하다.

그리고 추가적인 기능이 필요한데 예를 들어 p와 q가 임의 접근 반복자이고 n이 정수값이라면 

- p + n, p - n, n + p
- p - q
- p[n] : *(p + n)과 같은 의미
- p < q  ,p > q,  p <= q  ,p >= q : 비교 연산

2개의 반복자를 빼기 연산을 하면 두 반복자 사이의 거리를 정수 타입으로 반환한다.

임의 접근 반복자가 필요한 유일한 알고리즘은 sort 함수이다. 그리고 벡터와 문자열 반복자는 임의 접근 반복자이다.

하지만 리스트 반복자는 양방향 반복자만 지원하는데 list는 빠른 삽입과 삭제 작업에 최적화 되었기 때문이다.

따라서 리스트의 임의 요소에 접근하는 방법은 존재하지 않는다. 리스트를 탐색하는 유일한 방법은 순차적으로 요소 각각을 살펴보는 것이다.



### 8.2.7 반복자의 범위

STL의 알고리즘은 보통 범위를 이렇게 받는다.

```c++
[begin, end)
```

- begin -> 시작 위치
- end -> **마지막 바로 다음 위치**

왜 마지막 다음위치를 쓰는 이유는 3가지가 있다.



첫번째 이유는 만약 마지막 요소를 end로 쓴다면 **끝인지 확인하려면 마지막 값이 뭔지 알아야 한다.**

그런데 만약 **값이 중복되어 있다면** 비교가 불가능 하다. 그래서 위치로 끝을 표현 한다.



두번째는 만약 데이터가 없다면 즉,

```c++
[]  //(빈 컨테이너)
```

마지막 요소 자체가 없다, 하지만 이렇게 하면 해결된다 

```c++
begin == end
```

즉 빈 범위는 begin과 end가 같다는 의미이다. 결국 빈 범위도 사용할 수 있게 된다.



마지막으로 반복자의 비교를 ==, !=만으로도 해결할 수 잇다.

```c++
while (begin != end) {
    // 처리
    ++begin;
}
```

<, >없이도 범위를 처리할 수 있다. 

만약 [begin, end] 구조라면 위의 코드에서는 마지막 요소를 처리하지 못한다. 처리하려면 <=를 사용해야 하는데

<, >는 모든 iterator에서 지원하지 않는다.

즉 단순한 비교 ==, !=만으로 범위를 안전하고 일관되게 처리할 수 있다.



## 8.3 입력 및 출력 반복자

표준 컨테이너는 입력 및 출력 반복자와 순방향 반복자를 구분할 필요는 없는데 구분되어 있다.

그 이유는 모든 반복자가 컨테이너와 직접 연결되어 있지 않기 때문이다.

예를 들어, 어떤 컨테이너 c가 push_back을 지원한다고 하자.
 이때 back_inserter(c)는 값을 컨테이너에 추가하는 역할을 하는 반복자인데, 이 반복자는 다른 반복자 기능은 지원하지 않고 **오직 output iterator의 조건만 만족한다.**



또다른 예로는 표준 라이브러리는 입출력 스트림과 연결되는 반복자도 제공한다.

- 입력 스트림과 연결된 반복자는 → input iterator
- 출력 스트림과 연결된 반복자는 → output iterator

이러한 반복자를 사용하면 일반적인 iterator 연산만으로도 스트림과 데이터를 주고 받을 수 있다.

예를 들어 

- istream_iterator는 현재 입력 위치에서 값을 읽어온다
- ostream_iterator는 현재 출력 위치에 값을 쓴다

이 스트림 반복자들은 <iterator> 헤더에 정의되어 있다.



### 8.3.1 입력 스트림 반복자

입력 스트림 반복자는 istream_iterator에 정의되어 있다.

```c++
vector<int> v;

// 표준 입력에서 int를 읽어서 v에 추가
copy(istream_iterator<int>(cin),
     istream_iterator<int>(),
     back_inserter(v));
```

이 코드에서 copy 함수의 처음 두 인자는 복사할 범위를 의미한다.

- 첫 번째 인자 → 시작 위치 (cin에서 읽기 시작)
- 두 번째 인자 → 끝 위치

여기서 두번째 인자는 istream_iterator<int>()는 어떤 파일에도 연결되지 않은 빈 iterator인데 이 iterator의 특징이 있다.

입력이 끝났거나 오류가 발생하면 이 값과 같아진다.

즉, 이 iterator는 입력의 끝을 나타내는 역할을 한다. 그래서 우리는 이 iterator를 이용해 입력이 끝날 때까지 계속 데이터를 읽을 수 있다.



여기서 중요한 건 C++에서는 input과 output 연산을 할 때 읽거나 쓸 데이터 타입을 반드시 명시해야 한다.

```c++
getline(cin, s);        // 문자열 읽기
cin >> s.name >> s.midterm >> s.final; // 문자열 + double 읽기
```

그래서 스트림 iterator도 템플릿 형태로 되어 있고 어떤 타입을 다룰지 명시해야 한다.



### 8.3.2 출력 스트림 반복자

출력 작업에는 ostream_iterator를 사용해야 한다.

```c++
copy(v.begin(), v.end(),
     ostream_iterator<int>(cout, " "));
```

이 코드는 vector의 모든 값은 공백으로 구분해서 출력한다.

ostream_iterator<int>(cout, " "));에서 " "는 각 값 뒤에 붙일 문자열이다.



결론적으로 

- istream_iterator  → input iterator의 한 종류
- ostream_iterator  → output iterator의 한 종류 

즉 입출력용 iterator의 구체적인 구현이다.

```
input iterator  → 읽기 기능
   ↑
istream_iterator → cin에서 읽는 실제 구현

output iterator → 쓰기 기능
   ↑
ostream_iterator → cout으로 쓰는 실제 구현
```



## 8.4 융통성을 위한 반복자 사용

6.1.1 에서 살펴본 split함수를 개선해보다. split함수는 vector<string>을 반환하는데 이를 벡터가 아닌 list<string>이나 또 다른 종류의 컨테이너로 반환할 때를 생각해 보자. 



값을 반환하는 대신 출력 반복자를 이용하여 split 함수를 작성할 수 있다.

해당 버전의 split 함수에서는 반복자를 사용하여 찾은 단어들을 출력한다. split 함수를 호출하는 시점에서는 값들을 저장할 출력 지점에 반복자를 바인딩 시킨다.

```c++
template <class Out>
void split(const string& str, Out os)
{
    typedef string::const_iterator iter;
    iter i = str.begin();

	while (i != str.end()) {
        // 선행 공백 무시
        i = find_if(i, str.end(), not_space);

        // 다음 단어의 끝 찾기
		iter j = find_if(i, str.end(), space);

		// [i, j) 범위의 문자들을 복사
        if (i != str.end())
			*os++ = string(i, j);
        i = j;
    }
}
```

split 함수는 반환 할 값이 없으므로 void가 반환 타입이다. 이 때 출력 반복자를 암시하는 타입 매개변수 Out가 있는 템플릿 함수로 split 함수를 만들었다.

매개변수 os의 타입은 Out이고 단어를 발견할 때 단어를 write하려고 os를 사용한다.

이 함수는 문자열을 단어로 나누어서 결과를 어딘가에 출력하는 함수인데 중요한 점은 어디에 저장할 지 모른다는 것이다. 그래서 iterator로 받는다.

기존 split은 vector<string>을 반환했지만 지금은 Out os로 반환한다. 이 의미는 결과가 vector일 수도 있고 list일 수도 있고 cout으로 출력할 수도 있다.



```c++
*os++ = string(i, j);
```

이 표현식을 보면 string(i, j)는 [i, j)의 범위에 하나의 단어를 생성하고, 

*os = 값은 출력 iterator에 값을 쓴 후 os++로 다음 위치로 이동한다.

즉 단어를 하나 출력하고, 다음으로 이동한다. 여기서 중요한 건 os가 무엇이냐에 따라 결과가 달라진 다는 것이다.



벡터에 저장할 수 있고

```c++
vector<string> v;
split(s, back_inserter(v));
```



리스트에 저장할 수 있고

```c++
list<string> word_list;
split(s, back_inserter(word_list));
```



바로 출력할 수 있다

```c++
split(s, ostream_iterator<string>(cout, "\n"));
```



main 함수를 보자

```c++
int main()
{
    string s;
    while (getline(std::cin, s)) 
		split(s, std::ostream_iterator<string>(std::cout, "\n"));
}
```

여기서는 split 함수를 호출하여 입력한 문자열을 단어 각각으로 구분하고 단어들을 표준 출력으로 출력한다.

이때 cout에 바인딩된 ostream_iterator<string>을 split 함수에 전달하여 출력한다.

split 함수에서 *os에 값을 할당하면 cout을 이용해 출력한다.



## 8.5 핵심 정리

**템플릿 함수**

```c++
template <class T1, class T2, ...>
return_type function_name(parameter_list);
```

각 타입 매개변수는 함수 정의 안에서 타입이 필요한 곳 어디에서든 사용할 수 있는 이름이다.

이 이름들은 함수의 매개변수 목록에 사용되어 하나 이상의 매개변수 타입을 지정해야 한다.



만약 모든 타입이 인자 목록에 나타나지 않는다면 호출한 지점에서 실제 타입으로 함수 이름을 정규화 해야 한다.

예를 들어

```c++
template<class T> T zero() { return 0; }
```

이런 함수는 호출할 때는 다음처럼 반환 타입을 명시적으로 제공해야 한다.

```c++
double x = zero<double>();
```



typename 키워드는 템플릿의 타입 매개변수로 정의된 타입을 사용하는 선언문을 정규화 하는데 사용해야 한다. 예를 들어

```c++
typename T::size_type name;
```

이 선언은 name이 size_type 타입을 가진다는 것을 의미하며 이 size_type은 반드시 T 내부에 정의된 타입이어야 한다.



**반복자**

C++ 표준 라이브러리의 가장 중요한 개념은 반복자를 알고리즘과 컨테이너 사이의 연결고리로 사용하여 알고리즘이 데이터 구조 독립성을 확보할 수 있다는 것이다.

또한 사용하는 반복자에 필요한 연산에 따라 알고리즘을 구분할 수 있다는 것은 컨테이너와 해당 컨테이너에 사용할 수 있는 알고리즘을 손쉽게 짝지을 수 있음을 의미한다

반복자 카테고리는 다섯 가지 종류가 있다. 나중 카테고리는 이전 카테고리의 연산을 포함한다.

- **입력 반복자**
   한 방향으로 순차 접근, 읽기만 가능
- **출력 반복자**
   한 방향으로 순차 접근, 쓰기만 가능
- **순방향 반복자**
   한 방향으로 순차 접근, 읽기와 쓰기 모두 가능
- **양방향 반복자**
   양방향으로 순차 접근, 읽기와 쓰기 모두 가능
- **임의 접근 반복자**
   임의 위치에 효율적으로 접근 가능, 읽기와 쓰기 모두 가능
