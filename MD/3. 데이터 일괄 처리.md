# 3. 데이터 일괄 처리

학생의 시험 및 과제 점수를 입력받고 최종 점수를 계산하는 프로그램을 작성해보자

## 3.1 학생의 최종 점수 계산하기

한 과목에서 중간시험 20%, 기말 시험 40%, 평균 과제 점수 40%에 가중치를 주어 최종 점수를 구하는 프로그램을 만들어 보자

```c++
#include <iomanip>
#include <ios>
#include <iostream>
#include <string>

using std::cin; using std::setprecision;
using std::cout; using std::string;
using std::endl; using std::streamsize;


int main()
{
    // 학생의 이름을 묻고 입력받음
    std::cout << "Please enter your first name : ";
    string name;;
	cin >> name;
	cout << "Hello, " << name << "!" << endl;

	// 중간고사와 기말고사 성적을 묻고 입력받음
	cout << "Please enter your midterm and final exam grades : ";
	double midterm, final;;
	cin >> midterm >> final;

	// 과제 점수를 물음
	cout << "Enter all your homework grades, "
		<< "followed by end-of-file: ";

	// 지금까지 입력된 과제 점수의 개수와 합
	int count = 0;
	double sum = 0;

	// 입력을 위한 변수
	double x;

	// 불면성 : 지금까지 count개 점수를 입력받았으며 입력받은 점수의 합은 sum
	while (cin >> x) {
		++count;
		sum += x;
	}

	streamsize prec = cout.precision();
	cout << "Your final grade is " << setprecision(3)
		<< 0.2 * midterm + 0.4 * final + 0.4 * sum / count
		<< setprecision(prec) << endl;

	return 0;
}

```

<ios>는 입출력 라이브러리에서 전송된 문자 수나 버퍼 크기를 나타내려고 사용하는 타입인 streamsize를 정의한다

<iomanip>은 출력 결과의 유효 자릿수를 결정하는 조작어인 setprecision을 정의한다.

1. 이름, 중간고사 및 기말고사 점수를 묻고 입력 받는다.
2. 과제 점수를 묻고 EOF 신호를 만날 때까지 입력받는다 ( 새로운 행 입력, ctrl + Z , ctrl + D 등 )
3. 입력 맏는 동안 count를 사용해서 점수 개수 추적 및 누적 점수 sum 저장



```c++
cout << "Please enter your midterm and final exam grades : ";
double midterm, final;;
cin >> midterm >> final;
```

double 타입은 8byte(64bit)의 정밀도인 부동소수점을 나타내는 기본 타입



입력 연산자는 왼쪽 피연산자를 결과로 반환한다. 따라서 출력 작업을 연결한 것처럼 입력 작업을 연결할 수 있다.

```c++
cin >> midterm >> final;
```

이 코드는 다음과 같다

```c++
cin >> midterm;
cin >> final;
```



그 다음 코드는 과제 점수를 입력 받는다.

```c++
cout << "Enter all your homework grades, "
	<< "followed by end-of-file: ";
```

이 부분을 보면 2개의 문자열 리터럴을 출력하지만 <<는 하나이다.

공백으로 분리된 2개 이상의 문자열 리터럴은 자동으로 병합된다

다음과 같은 의미이다

```
cout << "Enter all your homework grades, followed by end-of-file: ";
```



다음은 입력받은 정보를 유지하는데 사용할 변수를 정의한다

```c++
int count = 0;
double sum = 0;
```

count와 sum의 초기값은 0이다. 0이란 값은 int타입이다.

sum을 초기화 하려면 0을 double 타입으로 변환해야 한다. 구현체는 컴파일 중에 변환을 시도하므로 런타임 오버헤드가 발생하지 않는다.

변수의 초기값을 지정 하지 않으면 **기본 초기화** 단계를 따르고, 변수 타입에 따라 결과가 다르다.

명시적으로 초기화 하지 않은 지역 변수는 **정의하지 않은 상태**이며 생성된 메모리 공간을 이미 차지하는 의미 없는 값이다.



while문에 새롭게 살펴볼 것은 조건문의 형태이다.

```c++
// 불면성 : 지금까지 count개 점수를 입력받았으며 입력받은 점수의 합은 sum
while (cin >> x) {
    ++count;
    sum += x;
}
```

while 문의 조건인 cin >> x 에서 입력이 발생하는 한 계속 실행된다.

새로운 입력 요청이 발생하면 조건을 만족시킨다.





결과를 출력하는 코드를 보면

```c++
streamsize prec = cout.precision();
cout << "Your final grade is " << setprecision(3)
	<< 0.2 * midterm + 0.4 * final + 0.4 * sum / count
	<< setprecision(prec) << endl;
```

목표는 최종 점수를 3자리 숫자로 출력하는 것이고 setprecision을 사용했으며 setprecision은 조작어이다.

setprecision은 이어서 발생하는 출력을 주어진 숫자만큼의 유효 자릿수로 나타내려고 스트림을 조작한다.

setprecision(3)이라고 작성하면 점수를 3개의 유효자리수로 출력하도록 구현체에게 요청하라는 의미고 일반적으로 소수점 앞 두 자리, 소수점 뒤 한 자리를 출력한다.



이후에 다시 setprecision을 사용해 cout에 있을지 모르는 후속 출력에 정밀도를 바꾼다.

cout에 멤버 한수 precision을 호출하면 부동 소수점 표현에 사용하는 정밀도를 알려준다. 

즉, setprecision으로 정밀도를 3으로 바꾼 다음 최종 점수를 출력하고 precision함수가 알려준 값으로 정밀도를 다시 바꾼다.



### 3.1.1 입력 종료 판별

while문의 조건에서 조건의 암묵적인 대상은 istream이다.

```c++
while (cin >> x) {}
```

먼저 cin을 이용해 입력을 받고 입력받은 값을 x에 넣으면 while문이 실행된다. 입력이 일어나지 않거나 x의 타입과 맞지 않는 값이 입력된다면 while문은 실행하지 않는다. x값이 필요한 상황이 사라진 것이다.

'>>' 연산자가 왼쪽 피연산자를 반환한다는 점을 기억하자. 즉 cin >> x 의 값을 요청하는 것은 cin >> x를 실행하고 다음 cin 값을 요청하는 것 과 같다

다음 코드를 보자

```c++
if (cin >> x){}
```

앞 코드는 다음과 같은 의미이다

```c++
cin >> x;
if (cin){}
```

cin >> x를 조건으로 사용하는 것은 조건을 판별하는 것만이 아니라 입력받은 값을 x에 넣는 부수적인 효과를 얻는다.

2장에서 다룬 조건은 모두 bool 타입의 값을 직접 반환하는 관계연산자가 있고, 숫자 값을 반환하는 표현식을 사용한다.

조건에서는 해당 값이 bool로 바뀐다. 0이 아닌 값은 true, 0인 값을 false로 바뀐다.

istream 클래스에서는 cin을 숫자값으로 바꿀 수 있다 또한 해당 숫자 값을 bool 타입 값으로 바꿔 조건 판별에 사용한다.

따라서 조건 판별에서 cin을 사용하는 것은 cin을 이용해 최근 입력을 받았는지 판별하는 것과 같다.

스트림에서 입력값을 읽을 수 없는 상황은 다음과 같다.

- EOF에 도달했을 때
- 변수의 타입과 맞지 않는 입력을 받았을 때
- 시스템이 입력장치에서 하드웨어 장애를 감지했을 때

이와 같은 상태의 입력스트림을 조건 판별에 사용하면 거짓이 된다.



### 3.1.2 루프 불변성

cin >> x가 실행되면 불변성의 첫번째 부분(지금까지 count개 점수를 입력받음)은 거짓이 된다. 그래서 count + 1을 해서 불변성을 참으로 만든다. 

그 다음에 불변성의 두번째 부분(입력 받은 점수의 합은 sum)이 거짓이 되기에 sum += x;를 실행하여 불변성을 참으로 만든다.



## 3.2 평균값 대신 중앙값 사용하기

과제 점수를 입력받고 즉시 버리는 것은 평균값을 계산할 때는 문제가 없으나 중앙 값을 사용할 때는 문제가 생긴다.

중앙 값을 찾는 방법은 오름차순이나 내림차순으로 정렬한 후 가운데 값을 선택하는 것이도 값이 짝수라면 두 값의 평균을 취하면 된다.



### 3.2.1 데이터 집단을 벡터에 저장하기

중앙 값을 사용할면

- 한번에 하나씩 입력받고 저장한다.
- 입력이 끝나면 값을 정렬
- 가운데 위치한 값 찾기

이를 효율적으로 하기 위헤 표준 라이브러리는 **벡터 타입**을 제공한다.

벡터타입으로 받도록 수정하면

```c++
// 입력을 위한 변수
double x;
vector<double> homework;

// 불면성 : homework는 지금까지 입력받은 과제 점수가 있음
while (cin >> x) {
    homework.push_back(x);
}
```

homework는 vector<double>타입으로 정의된다.

벡터는 값의 집단이 있는 **컨테이너**이며 각 벡터의 값은 모두 같은 타입이다.

벡터를 정의할 때마다 벡터가 담을 값의 타입을 지정해야 하며 homework의 정의를 살펴보면 double값을 담을 벡터라는 것을 알 수 있다.

벡터 타입은 C++에 특징인 **템플릿 클래스**로 정의한다. 벡터와 벡터에 담은 특정 타입의 객체를 의미상 구분하는 것으로 객체의 타입은 꺽쇠괄호를 사용해서 지정한다.

예를 들어 vector <double>타입의 객체는 double 타입의 객체를 담을 벡터이고, vector<string>타입의 객체는 문자열을 담은 벡터이다.



```c++
homework.push_back(x);
```

push_back은 멤버함수이다. push_back은 벡터 타입의 일부로 정의하며, 객체 homework와 관련된 작업을 실행할 수 있다.

push_back은 벡터의 마지막에 새 요소를 추가하는 작업을 실행하며 인수로 전달한 값을 넣는다.

push_back은 해당 인수를 백터의 뒤쪽으로 밀어내며, 벡터의 크기는 1씩 증가한다.



### 3.2.2 출력 생성하기

중앙 값을 찾으려면 homework의 크기를 최소 2번 알아야 한다. 첫번째는 벡터 크기가 0인지 확인할 때, 두번째는 가운데 요소의 위치를 계산하려 할때 사용한다.

크기를 알아보는 과정을 반복하지 않으려고 지역 변수에 벡터의 크기를 저장한다. 

```c++
typedef vector<double>::size_type vec_sz;
vec_sz size = homework.size();
```

size_type은 크기가 큰 벡터도 담을 수 있는 부호 없는 타입이고 size()는 벡터의 요소 개수를 나타내는 size_type의 값을 반환한다.

vector<double>::size_type은 이름이 길어서 다루기 불편하다 이를 단순하게 하기 위해 typedef라는 언어 기능을 사용한다.

vector<double>::size_type 타입의 동의어로 vec_sz라는 이름을 지정한다.



비어있는 데이터터가 있는지 확인한다.

```c++
if (size == 0) {
    cout << endl << "You must enter your grades. "
        << "Please try again." << endl;
    return 1;
}
```

size가 0이면 문제가 있음을 알리고 프로그램을 종료해야 한다. 문제가 있다면 1을 반환한다.

시스템은 main함수에서 0을 반환하면 프로그램이 정상적으로 동작했다고 간주한다.



데이터가 있다면 중앙 값을 계산하자. 라이브러리를 호출해서 데이터를 정렬해야 한다.

```c++
sort(homework.begin(), homework.end());
```

<algorithm>으로 정의하는 sort 한수는 컨테이너가 담은 값들을 비내림차순으로 다시 정리한다.

sort함수의 인수로는 정렬할 요소의 범위를 지정한다. 이때 벡터 클래스에서 제공하는 2개의 멤버 함수 begin과 end를 사용한다.

homework.begin()은 첫번째 요소, homework.end()는 마지막 요소를 의미한다.

sort 함수는 정렬한 결과를 담으려고 새 컨테이너를 만들지 않는다. 단지 컨테이너의 요소 값들을 이동하여 정렬한다.



정렬했다면 가운데 요소의 위치를 찾아야 한다.

```c++
vec_sz mid = size / 2;
double median;
median == size % 2 == 0 ? (homework[mid] + homework[mid - 1]) / 2 : homework[mid];
```

가운데 요소를 찾기 위해 size를 2로 나눈다.

요소 개수가 짝수일 경우 정수지만 홀수라면 결과의 소수점을 내림한 정수이다.

median에서 값을 할당하는 표현식에서 2개의 새로운 연산자를 사용한다.

%는 나머지 연산자이고 ?는 조건 연산자이다.

% 연산자를 사용해 2로 나눈 나머지가 0이면 짝수개의 점수를 입력받은 것이다.

? 연산자는 if - then - else를 줄인 것으로 

?앞의 표현식 size % 2 == 0 조건을 판별에 bool값 반환하고 참이면 ?과 : 사이의 표현식을 반환하고 그렇지 않으면 : 뒤에 표현식을 반환한다.

짝수라면 가운데 두 요소 중 평균을 median을 할당하고 홀수라면 home[mid]를 median에 할당한다.

homework[mid]와 homework[mid-1]은 벡터 요소의 접근하는 방법이다.

첫번째 요소는 homework[0]이고 마지막 요소는 homework[size -1]이다.

각 요소 자체는 컨테이너에 저장된 이름 없는 객체이며 homework[mid]는 double타입의 객체이며 double 타입을 지원하는 노든 연산을 사용할 수 있다.



size가 짝수면 mid는 size / 2로 정확히 계산할 수 있고 가운데 요소 2개가 필요하다.

가장 가까운 요소가 인덱스 mid - 1과 mid임으로 중앙 값은 이 두 요소의 평균 값이며 



size가 홀수이면 2로 나누었을 때 버리는 부분이 생기므로 mid는 (size -1) /2와 같다.



최종 점수를 구해서 출력해보면 

```c++
streamsize prec = cout.precision();
cout << "Your final grade is " << setprecision(3)
    << 0.2 * midterm + 0.4 * final + 0.4 * median
    << setprecision(prec) << endl;
```



최종 프로그램은

```c++
#include <algorithm>
#include <iomanip>
#include <ios>
#include <iostream>
#include <string>
#include <vector>

using std::cin; using std::sort;
using std::cout; using std::streamsize;
using std::endl; using std::string;
using std::setprecision; using std::vector;


int main()
{
    // 학생의 이름을 묻고 입력받음
    std::cout << "Please enter your first name : ";
    string name;;
	cin >> name;
	cout << "Hello, " << name << "!" << endl;

	// 중간고사와 기말고사 성적을 묻고 입력받음
	cout << "Please enter your midterm and final exam grades : ";
	double midterm, final;;
	cin >> midterm >> final;

	// 과제 점수를 묻고 입력 받음
	cout << "Enter all your homework grades, "
		<< "followed by end-of-file: ";
	vector<double> homework;

	double x;

	// 불면성 : homework는 지금까지 입력받은 과제 점수가 있음
	while (cin >> x) {
		homework.push_back(x);
	}

	// 과제 점수와 입력 유무를 확인
	typedef vector<double>::size_type vec_sz;
	vec_sz size = homework.size();;
	if (size == 0) {
		cout << endl << "You must enter your grades. "
			<< "Please try again." << endl;
		return 1;
	}

	// 점수를 정렬
	sort(homework.begin(), homework.end());
	
	// 과제 점수의 중앙값을 구함
	vec_sz mid = size / 2;
	double median;
	median == size % 2 == 0 ? (homework[mid] + homework[mid - 1]) / 2 : homework[mid];

	// 최종 성적을 계산하여 출력
	streamsize prec = cout.precision();
	cout << "Your final grade is " << setprecision(3)
		<< 0.2 * midterm + 0.4 * final + 0.4 * median
		<< setprecision(prec) << endl;

	return 0;
}

```



### 3.2.3 추가 사항 

homework에 요소가 없을 때 프로그램을 종료하는 이유는 요소가 없는 집단의 중앙 값은 그 결과가 무엇을 의미하는지 알 수 없으므로 정의할 수 없기 때문이다.

입력값이 전혀 없고 최소한 1개의 숫자를 읽었음을 판별하는 과정도 없다면 코드는 시행되지 않는다 그 이유는

1. 입력값이 없다면 homework.size()는 0이며 size또한 0이다. 따라서 mid도 0이다.
2. homework[mid]를 실행하면 첫번째 요소를 찾는다. 하지만 homework에는 요소가 없으므로 의미가 없다.



vector<double>::size_type은 **부호 없는 정수 타입**이다. 음수 값을 저장할 수 없다.

표현식에 일반 정수와 부호 없는 정수가 하나의 표현식에 섞일 때는 일반 정수가 부호 없는 정수타입으로 바뀐다

homework.size() - 100일 경우 size가 100보다 작아도 음수일 수 없다.



마지막으로 vector<double>객체가 정확한 크기로 즉시 할당되지 않는다.

입력 받은 값을 처리하는데 필요한 만큼 크기가 커진다.



표준을 준수하는 모든 C++ 구현체는 다음 사항을 지켜야 한다.

- 벡터의 많은 개수의 요소를 추가할 때 성능이 요소 개수에 비례해서 나빠지지 않게 벡터를 구현해야 한다.
- sort 함수의 성능이 nlog(n)보다 느리지 않도록 구현해야 한다. 여기서 n은 정렬할 요소의 개수이다.

이에 따라 표준을 준수하는 모든 구현체에서 보통 nlog(n) 시간 또는 그보다 이른 시간 안에 실행할 수 있다.



## 3.3 핵심 정리

**지역 변수**

지역 변수는 명시적으로 초기화 하지 않고 정의했다면 "기본 초기화" 된다. 기본 타입의 "기본 초기화"는 값을 정의하지 않았다는 것이며 왼쪽 피연산자로만 사용할 수 있다.



**타입 정의문**

타입 정의문은 typedef 타입 이름; 처럼 '이름'을 '타입'의 동의어로 정의한다.

<vector>헤더에 정의된 벡터 타입은 지정된 타입 값의 순차열을 저장하는 컨테이너인 라이브러리 타입이며 동적으로 크기가 늘어난다.

- vector<T>::size_type : 최대 크기의 벡터에 있는 요소 개수를 저장한다.
- v.begin() : 첫번째 요소가 가리키는 값을 반환한다.
- v.end() : 마지막 요소 다음을 가리키는 값을 반환한다.
- vector<T>v; : T타입의 요소를 지정할 수 있는 빈 벡터를 만든다.
- v.push_back(e) : e로 초기화된 하나의 요소를 벡터에 추가한다.
- v[i] : i 위치에 저장된 값을 반환한다.
- v.size() : v에 있는 요소 개수를 반환한다.



**다른 라이브러리 기능**

- sort(b,e) : [b,e) 범위에 있는 요소들을 오름차순으로 다시 정렬한다. <algorithm>헤더에 정의되어 있다.
- max(e1, e2) : e1과 e2 중 더 큰 것을 반환한다. 두 요소는 정확히 같은 타입이어야 한다. <algorithm>헤더에 정의되어 있다.
- while(cin >> x) : x에 적절한 타입의 값을 읽어들여 스트림의 상태를 판별한다. 만약 스트림이 오류상태면 while문의 조건은 거짓, 그렇지 않으면 while 본문을 실행한다.
- s.precision(n) : 스트림 s에 출력 정밀도를 n으로 설정하고 n을 생략하면 정밀도를 바꾸지 않는다. 이전 정밀도를 반환한다.
- setprecision(n) : 출력 스트림 s로 출력할 때 s.precision(n)을 호출과 같은 효과를 갖는 값으로 반환한다. <iomanip>에 정의되어 있다.
- streamsize: setprecision함수와 precision 함수가 반환하는 값의 타입이다. <ios>헤더에 정의되어 있다.
